# v1.0 Requirements — CST-to-AST Lowering Pipeline

**Milestone:** v1.0
**Scope:** Complete lowering pipeline transforming Writ CST to simplified AST
**Source:** Language spec §13–§20, §28; project research `.planning/research/`

## Success Criteria

The pipeline is complete when:
1. Every Writ construct that has a lowering rule in spec §28 is correctly desugared
2. The lowered AST contains no CST-specific sugar — only functions, structs, dispatch table entries, and primitives
3. Every AST node carries a `SimpleSpan` pointing to its CST origin for downstream error reporting
4. Lowering errors (unknown speakers, invalid transitions, key collisions) are reported with source locations
5. Adding a new desugaring pass requires creating one file and adding one call — no restructuring

## Requirements

### R1 — AST Type Hierarchy

**Priority:** P1 (blocks everything)

Define a simplified AST node set in `writ-compiler` that is the sole output of the lowering pipeline and sole input to all downstream compiler phases.

**Acceptance:**
- [x] `AstExpr`, `AstStmt`, `AstDecl`, `AstType` enum types defined
- [x] Every node carries `span: SimpleSpan` — no exceptions
- [x] Owned types only (`String`, `Box<T>`, `Vec<T>`) — no `'src` lifetime from CST
- [x] No CST sugar variants (`Dlg`, `Entity`, `T?`, `FormattableString`, compound assignments) — those are lowered away before reaching AST
- [x] Concurrency primitives (`Spawn`, `Join`, `Cancel`, `Defer`) represented as AST-level nodes (pass-through, not desugared)
- [x] `Expr::Error` / `Stmt::Error` variants preserved for error recovery continuity

### R2 — Pipeline Infrastructure

**Priority:** P1 (blocks all passes)

Provide a `LoweringContext` struct and pipeline orchestrator that sequences passes and accumulates errors without halting.

**Acceptance:**
- [x] `LoweringContext` carries `errors: Vec<LoweringError>`, `speaker_stack: Vec<SpeakerScope>` (loc_key_counter removed in Phase 7 — replaced by fnv1a_32() in dialogue.rs)
- [x] Public API: `lower(items: Vec<Spanned<Item<'_>>>) -> (Ast, Vec<LoweringError>)`
- [x] Errors from any pass do not prevent other passes from running
- [x] `LoweringError` type carries source `SimpleSpan` and descriptive message
- [x] Pass ordering documented in `lower/mod.rs` with rationale

### R3 — Optional Sugar Lowering

**Priority:** P1 (foundational — affects all type positions)

Desugar nullable/optional shorthand per spec.

**Acceptance:**
- [x] `T?` → `Option<T>` in all type positions (params, returns, fields, locals)
- [x] `null` literal → `Option::None`
- [ ] `?` propagation → early return on `None`
- [ ] `!` unwrap → panic on `None`

### R4 — Formattable String Lowering

**Priority:** P1 (required by dialogue lowering)

Desugar interpolated strings per spec §28.1.

**Acceptance:**
- [x] `$"Hello {name}!"` → `"Hello " + name.into<string>() + "!"`
- [x] Nested expressions in `{expr}` handled recursively
- [ ] Escaped braces (`{{`, `}}`) produce literal `{`, `}` — needs snapshot test verification
- [ ] Dialogue text lines (implicitly formattable per §13.9) lowered the same way

### R5 — Compound Assignment Desugaring

**Priority:** P1 (mechanical, required before operator lowering is complete)

Desugar compound assignment operators per spec §17.3.

**Acceptance:**
- [x] `a += b` → `a = a + b` (uses `Add`)
- [x] `a -= b` → `a = a - b` (uses `Sub`)
- [x] `a *= b` → `a = a * b` (uses `Mul`)
- [x] `a /= b` → `a = a / b` (uses `Div`)
- [x] `a %= b` → `a = a % b` (uses `Mod`)

### R6 — Operator Lowering

**Priority:** P1

Desugar operator overload declarations to contract implementations per spec §17.2.

**Acceptance:**
- [x] `operator +(other: T) -> T { ... }` inside `impl` → `impl Add<T, T> for Self`
- [x] Binary operators: `+`, `-`, `*`, `/`, `%`, `==`, `<`
- [x] Unary operators: `-()`, `!()` → `Neg`, `Not` contract impls
- [x] Index operators: `[]` → `Index`, `[]=` → `IndexMut`
- [x] Derived operators auto-generated per spec §17.4:
  - `a != b` → `!(a == b)` (from `Eq`)
  - `a > b` → `b < a` (from `Ord`)
  - `a <= b` → `a < b || a == b` (from `Eq` + `Ord`)
  - `a >= b` → `!(a < b)` (from `Eq` + `Ord`)

### R7 — Concurrency Pass-Through

**Priority:** P1

Ensure concurrency primitives survive lowering as first-class AST nodes.

**Acceptance:**
- [x] `spawn` (task creation) → `AstExpr::Spawn`
- [x] `join` → `AstExpr::Join`
- [x] `cancel` → `AstExpr::Cancel`
- [x] `defer` → `AstExpr::Defer`
- [x] `detached` → `AstExpr::Detached`
- [x] No transformation — CST node maps 1:1 to AST node with span preserved

### R8 — Dialogue Lowering

**Priority:** P1 (most complex pass)

Desugar `dlg` declarations to `fn` declarations per spec §28.1–§28.2.

**Acceptance:**
- [x] `dlg name(params) { body }` → `fn name(params) { lowered_body }`
- [x] `@Speaker text` → `say_localized(speaker_ref, key, "text")`
- [x] `@Speaker` (standalone) → sets active speaker for subsequent lines
- [x] Speaker resolution follows three tiers per spec §13.2:
  1. Local variables/parameters → use directly
  2. `[Singleton]` entities → `Entity.getOrCreate<T>()`
  3. Otherwise → `LoweringError` with span (TextLine with no active speaker only)
- [x] Active speaker scoped correctly across `$ choice` branches (push/pop, no leak)
- [x] `$ statement;` → `statement;` (code escape, no wrapping)
- [x] `$ { block }` → `{ block }` (code block escape)
- [x] `$ if cond { ... }` → `if cond { lowered_dialogue_branches }`
- [x] `$ match expr { ... }` → `match expr { lowered_dialogue_arms }`
- [x] `$ choice { ... }` → `choice([Option("label", fn() { ... }), ...])`
- [x] `-> target` → `return target()`
- [x] `-> target(args)` → `return target(args)`
- [x] `{expr}` in dialogue text → formattable string lowering (R4 pattern)

### R9 — Localization Key Generation

**Priority:** P1

Generate localization keys for dialogue text per spec §28.4.

**Acceptance:**
- [x] Auto-keys computed via FNV-1a 32-bit hash of `namespace\0method\0speaker\0content\0occurrence_index`
- [x] Manual `#key` overrides replace auto-generated keys
- [x] `say()` calls become `say_localized(speaker, key, fallback_text)`
- [x] Choice labels also support `#key` per spec §13.7
- [x] Keys deterministic across compiler runs (no pointer/allocation-order values)

### R10 — Localization Key Collision Detection

**Priority:** P1

Detect duplicate localization keys per spec §13.7.

**Acceptance:**
- [x] Duplicate `#key` values within a `dlg` block → `LoweringError::DuplicateLocKey`
- [x] Auto-generated key collisions detected and reported (occurrence_index disambiguation)
- [x] Error includes both conflicting line spans

### R11 — Dialogue Transition Validation

**Priority:** P1

Enforce `->` transition semantics per spec §13.6.

**Acceptance:**
- [x] `->` must be the last statement in its block → `LoweringError::NonTerminalTransition` if followed by more statements
- [x] Validates target exists (or defers to name resolution if lowering can't see cross-module)

### R12 — Entity Lowering

**Priority:** P1

Desugar `entity` declarations per spec §28.3.

**Acceptance:**
- [x] `entity Name { ... }` → `struct Name { component_fields, property_fields }`
- [x] `use Component { field: val, ... }` → component field with defaults + `ComponentAccess<T>` impl
- [x] Entity methods → `impl Name { methods }`
- [x] `on create { ... }` → lifecycle hook registration
- [x] `on interact(who) { ... }` → lifecycle hook registration
- [x] `on destroy { ... }` → lifecycle hook registration
- [x] `[Singleton]` attribute recognized and propagated
- [x] Member partitioning (properties vs. components vs. hooks vs. methods) is an explicit pre-step

### R13 — Component Field Flattening

**Priority:** P1

Merge component `use` clause field overrides with component defaults.

**Acceptance:**
- [x] `use Health { current: 80, max: 80 }` → `_health: Health` field with initializer `Health { current: 80, max: 80 }`
- [x] Fields not overridden use component's own defaults
- [x] Conflicting method names across components → `LoweringError` per spec §14.3

### R14 — Span Preservation

**Priority:** P1 (cross-cutting constraint)

Every AST node must carry source span information for downstream error reporting.

**Acceptance:**
- [x] Synthetic nodes (generated `say()` calls, `Entity.getOrCreate<T>()`, etc.) carry `lowered_from` span pointing to CST origin
- [x] No `SimpleSpan::new(0, 0)` tombstones on nodes that can produce errors
- [x] Downstream phases can report errors that point to original source locations

### R15 — Snapshot Testing

**Priority:** P1 (quality gate)

All lowering passes tested via `insta` snapshot tests.

**Acceptance:**
- [x] Each pass has snapshot tests covering its core transformations
- [x] Dialogue lowering tests cover: speaker resolution (all 3 tiers), `$ choice` scoping, `->` transitions, `#key` overrides
- [x] Entity lowering tests cover: component flattening, lifecycle hooks, `[Singleton]`
- [x] Integration test: full program with mixed constructs lowered end-to-end

## Non-Requirements

- Type checking / name resolution — separate compiler phase, not part of lowering
- Code generation — downstream of AST
- Runtime implementation — `writ-runtime` crate
- Macro system — not in current spec
- Optimization passes — premature
- Incremental compilation / Salsa — no performance need yet
- IDE integration / LSP — needs consumer infrastructure first

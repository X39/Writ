The AST types currently derive `Debug, Clone, PartialEq` but NOT `Serialize`. The research specifies `assert_ron_snapshot!` which requires `Serialize`. We have two options: use `assert_debug_snapshot!` (works with `Debug` already) or add `Serialize`. RON snapshots are significantly more readable and diff-friendly than Debug output for nested enums, so add `Serialize`.

**Step 1: Add serde dependency to Cargo.toml**

Add `serde` with `derive` feature to `[dependencies]` in `writ-compiler/Cargo.toml`:
```toml
serde = { version = "1", features = ["derive"] }
```

This is a production dependency (not dev-only) because `Serialize` must be derived on the AST types themselves, which are in `src/`. The `derive` feature enables `#[derive(Serialize)]`.

**Step 2: Add `Serialize` derive to all AST types**

```rust
#[test]
fn fmt_string_simple_interpolation() {
    // $"Hello {name}!" â†’ "Hello " + name.into<string>() + "!"
    let ast = lower_src(r#"fn f(name: string) { let x = $"Hello {name}!"; }"#);
    insta::assert_debug_snapshot!(ast);
}

#[test]
fn fmt_string_no_interpolation() {
    // $"plain string" â†’ StringLit (no concat chain needed â€” single Text segment)
    let ast = lower_src(r#"fn f() { let x = $"plain string"; }"#);
    insta::assert_debug_snapshot!(ast);
}

#[test]
fn fmt_string_multiple_segments() {
    // $"a={a} b={b}" â†’ "a=" + a.into<string>() + " b=" + b.into<string>()
    let ast = lower_src(r#"fn f(a: int, b: int) { let x = $"a={a} b={b}"; }"#);
    insta::assert_debug_snapshot!(ast);
}

#[test]
fn fmt_string_escaped_braces() {
    // $"{{literal}}" â†’ StringLit with "{literal}" (braces de-escaped by lexer)
    // Note: if the lexer does NOT de-escape {{ to {, the snapshot will show "{{literal}}"
    // and this test documents that behavior for correction in the lexer.
    let ast = lower_src(r#"fn f() { let x = $"{{literal}}"; }"#);
    insta::assert_debug_snapshot!(ast);
}
```

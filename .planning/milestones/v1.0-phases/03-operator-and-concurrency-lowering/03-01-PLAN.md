**1. Public function `lower_operator_impls`:**
```rust
pub fn lower_operator_impls(
    i: ImplDecl<'_>,
    i_span: SimpleSpan,
    ctx: &mut LoweringContext,
) -> Vec<AstDecl>
```

This function:
- Lowers `i.target` via `lower_type` to get `target_type: AstType`
- Lowers `i.contract` via `lower_type` to get `contract_type: Option<AstType>`
- Iterates over `i.members`, partitioning into `fn_members: Vec<AstImplMember>` and `operator_decls: Vec<AstDecl>`
- For each `ImplMember::Fn((fn_decl, fn_span))`: push `AstImplMember::Fn(lower_fn(fn_decl, fn_span, ctx))` to `fn_members`
- For each `ImplMember::Op((op_decl, op_span))`:
  - Call `op_to_contract_impl(&op_decl, op_span, &target_type, ctx)` to get an `AstDecl::Impl` with the correct contract type
  - Push to `operator_decls`
  - Track whether this operator was `Eq` or `Lt` (for derived generation)

After processing all members:
- If `fn_members` is non-empty OR `contract_type.is_some()`, emit a base impl: `AstDecl::Impl(AstImplDecl { contract: contract_type, target: target_type.clone(), members: fn_members, span: i_span })`
- If only operators and no contract — do NOT emit an empty base impl
- Append all `operator_decls`
- Call `generate_derived_operators(has_eq, has_ord, &target_type, i_span)` and append results

Return the combined `Vec<AstDecl>`.

**2. Private function `op_to_contract_impl`:**

Maps an `OpDecl` to an `AstDecl::Impl` node with:
- `contract: Some(AstType::Generic { name: contract_name, args: type_args, span: op_span })`
- `target: target_type.clone()`
- `members: vec![AstImplMember::Fn(op_body_as_fn)]`
- `span: op_span`

The operator body is lowered to an `AstFnDecl` with:
- `name`: the method name from the mapping table (see below)
- `params`: lowered from `op_decl.params` via `lower_param`
- `return_type`: lowered from `op_decl.return_type` via `lower_type`
- `body`: lowered from `op_decl.body` via `lower_stmt`
- `vis`: lowered from `op_decl.vis`
- `attrs`, `generics`: empty vecs (operators don't have these in the CST)
- `span`: `op_span`
- `name_span`: `op_decl.symbol.1` (the symbol's span serves as the name span)

**3. Private function `op_symbol_to_contract`:**

Complete mapping (use explicit match, NO `_ =>` wildcard):

| OpSymbol | Params | Contract Name | Type Args | Method Name |
|----------|--------|---------------|-----------|-------------|
| Add | 1 | "Add" | [param_type, return_type] | "add" |
| Sub | 1 | "Sub" | [param_type, return_type] | "sub" |
| Sub | 0 | "Neg" | [return_type] | "neg" |
| Mul | 1 | "Mul" | [param_type, return_type] | "mul" |
| Div | 1 | "Div" | [param_type, return_type] | "div" |
| Mod | 1 | "Mod" | [param_type, return_type] | "mod_op" |
| Eq | 1 | "Eq" | [param_type] | "eq" |
| Lt | 1 | "Ord" | [param_type] | "lt" |
| Not | 0 | "Not" | [return_type] | "not" |
| Index | 1 | "Index" | [param_type, return_type] | "index" |
| IndexSet | 2 | "IndexMut" | [param1_type, param2_type] | "index_set" |

**CRITICAL — Sub disambiguation:** `OpSymbol::Sub` with zero params = unary `Neg`; with one param = binary `Sub`. Use a match guard or check `op.params.is_empty()` before the main symbol match.

For `Eq` and `Ord`: Use single type arg `[param_type]` — return type `bool` is implied by the contract.

For `IndexMut`: Two type args `[index_param_type, value_param_type]`. The OpDecl for `[]=` has 2 params.

Type args are constructed by calling `lower_type` on each param's type from `op_decl.params`.

**4. Private function `generate_derived_operators`:**

Takes `has_eq: bool`, `has_ord: bool`, `eq_param_type: Option<&AstType>`, `ord_param_type: Option<&AstType>`, `target_type: &AstType`, `impl_span: SimpleSpan`.

Rules:
- If `has_eq`: Generate `impl Ne for target { fn ne(other: T) -> bool { !(self == other) } }`
  - Contract: `AstType::Generic { name: "Ne", args: [param_type], span: impl_span }`
  - Body: one `AstStmt::Return` containing `AstExpr::UnaryPrefix { op: PrefixOp::Not, expr: AstExpr::Binary { left: AstExpr::SelfLit, op: BinaryOp::Eq, right: AstExpr::Ident("other") } }`
  - Use `AstExpr::SelfLit { span: impl_span }` for `self` — NOT `AstExpr::Ident { name: "self" }`

- If `has_ord`: Generate `impl Gt for target { fn gt(other: T) -> bool { other < self } }`
  - Body: `AstStmt::Return` containing `AstExpr::Binary { left: AstExpr::Ident("other"), op: BinaryOp::Lt, right: AstExpr::SelfLit }`

- If `has_eq` AND `has_ord`: Generate two more:
  - `impl LtEq for target { fn lt_eq(other: T) -> bool { self < other || self == other } }`
    - Body: `AstExpr::Binary { left: Binary(SelfLit Lt Ident("other")), op: BinaryOp::Or, right: Binary(SelfLit Eq Ident("other")) }`
  - `impl GtEq for target { fn gt_eq(other: T) -> bool { !(self < other) } }`
    - Body: `AstExpr::UnaryPrefix { op: Not, expr: Binary(SelfLit Lt Ident("other")) }`

All synthetic nodes use `impl_span` for their span fields — never `SimpleSpan::new(0, 0)`.

Each derived impl is an `AstDecl::Impl(AstImplDecl { contract: Some(...), target: target_type.clone(), members: vec![AstImplMember::Fn(...)], span: impl_span })`.

The derived fn has: `name: "ne"|"gt"|"lt_eq"|"gt_eq"`, `params: vec![AstParam { name: "other", ty: param_type.clone(), ... }]`, `return_type: Some(AstType::Named { name: "bool", span: impl_span })`, `body: vec![the_return_stmt]`, `attrs: vec![]`, `generics: vec![]`, `vis: None`.

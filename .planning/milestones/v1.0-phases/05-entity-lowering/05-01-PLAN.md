**Step 1: Create entity.rs**

The file implements two functions:

**`partition_entity_members()`** — Pre-step that scans members into four buckets with validation:

```rust
fn partition_entity_members<'src>(
    members: Vec<Spanned<EntityMember<'src>>>,
    entity_name: &str,
    ctx: &mut LoweringContext,
) -> (
    Vec<(Option<Visibility>, Spanned<&'src str>, Spanned<TypeExpr<'src>>, Option<Spanned<Expr<'src>>>, SimpleSpan)>,  // properties
    Vec<(Spanned<&'src str>, Vec<Spanned<UseField<'src>>>, SimpleSpan)>,  // use_clauses
    Vec<(FnDecl<'src>, SimpleSpan)>,  // methods
    Vec<(Spanned<&'src str>, Option<Vec<Spanned<Param<'src>>>>, Vec<Spanned<Stmt<'src>>>, SimpleSpan)>,  // hooks
)
```

Alternatively, create small intermediate structs to avoid deeply nested tuples. Use whatever approach results in the clearest code.

Validation during partitioning (emit error, skip duplicate, continue):
- **Duplicate property names:** Track `seen_props: Vec<String>`. On duplicate → `LoweringError::DuplicateProperty`.
- **Duplicate use clauses:** Track `seen_components: Vec<String>`. On duplicate → `LoweringError::DuplicateUseClause`.
- **Property-component name collision:** If a use clause's component name matches a property name → `LoweringError::PropertyComponentCollision`. Note: check both directions — a property added AFTER a use clause should also collide. Track using both `seen_props` and `seen_components`, and check cross-set membership for both Property and Use members.
- **Unknown lifecycle events:** Valid events are `"create"`, `"interact"`, `"destroy"`. Any other event name → `LoweringError::UnknownLifecycleEvent`. Skip the hook (do not add to hooks vec).

**`lower_entity()`** — Main function that returns `Vec<AstDecl>`:

```rust
pub fn lower_entity(
    entity: EntityDecl<'_>,
    entity_span: SimpleSpan,
    ctx: &mut LoweringContext,
) -> Vec<AstDecl>
```

Steps:
1. Extract entity name and name span.
2. Lower attributes via `super::lower_attrs(entity.attrs, ctx)` — `[Singleton]` passes through unchanged.
3. Lower visibility via `super::lower_vis(entity.vis)`.
4. Call `partition_entity_members(entity.members, &entity_name, ctx)`.
5. Build `Vec<AstDecl>` in this deterministic emission order:

**Emit 1: AstDecl::Struct** — Always emitted.
- `attrs`: from step 2 (carries [Singleton] if present)
- `vis`: from step 3
- `name`, `name_span`: from entity
- `generics`: `vec![]` (entities have no generics)
- `fields`: Property fields (lowered via `lower_struct_field` if EntityMember::Property has matching CST StructField shape; otherwise build AstStructField manually from vis/name/ty/default) PLUS component fields. For properties, use `super::lower_vis(prop.vis)`, `lower_type(prop.ty)`, and `lower_expr(prop.default)` for defaults. For each use clause, emit:
  ```rust
  AstStructField {
      vis: None,  // component fields are not user-accessible
      name: format!("${}", component_name),
      name_span: component_span,
      ty: AstType::Named { name: component_name.clone(), span: component_span },
      default: Some(AstExpr::StructLit {
          name: component_name.clone(),
          name_span: component_span,
          fields: use_fields.iter().map(|(uf, uf_span)| {
              (uf.name.0.to_string(), uf.name.1, lower_expr(uf.value, ctx))
          }).collect(),
          span: component_span,
      }),
      span: use_clause_span,
  }
  ```
- `span`: `entity_span`

**Emit 2: AstDecl::Impl (inherent, methods)** — Only if methods vec is non-empty.
- `contract: None` (inherent impl, no contract)
- `target`: `AstType::Named { name: entity_name.clone(), span: entity_name_span }`
- `members`: each method lowered via `super::lower_fn(fn_decl, fn_span, ctx)` wrapped in `AstImplMember::Fn(...)`
- `span`: `entity_span`

**Emit 3: AstDecl::Impl (ComponentAccess<T>)** — One per use clause, in source order.
For each `(component, fields, use_span)`:
- `contract`: `Some(AstType::Generic { name: "ComponentAccess".to_string(), args: vec![AstType::Named { name: component_name, span: comp_span }], span: comp_span })`
- `target`: `AstType::Named { name: entity_name.clone(), span: entity_name_span }`
- `members`: Single `AstImplMember::Fn(AstFnDecl { name: "get", params: vec![], return_type: Some(AstType::Named { name: component_name, span }), body: vec![AstStmt::Return { value: Some(AstExpr::MemberAccess { object: Box::new(AstExpr::SelfLit { span }), field: format!("${}", component_name), field_span: comp_span, span: comp_span }), span: comp_span }], span: use_span, ..defaults })`
- `span`: `use_span`

**Emit 4: AstDecl::Impl (lifecycle hooks)** — One per hook, in source order.
For each `(event, params, body, hook_span)`:
- Map event name to contract name and method name:
  - `"create"` → `OnCreate`, `on_create`
  - `"interact"` → `OnInteract`, `on_interact`
  - `"destroy"` → `OnDestroy`, `on_destroy`
- `contract`: `Some(AstType::Named { name: contract_name, span: event.1 })`
- `target`: `AstType::Named { name: entity_name.clone(), span: entity_name_span }`
- `members`: Single `AstImplMember::Fn(AstFnDecl { name: method_name, params: lower params via super::lower_param if present, return_type: None, body: body.into_iter().map(|s| lower_stmt(s, ctx)).collect(), span: hook_span, attrs: vec![], vis: None, generics: vec![], name_span: event.1 })`
- `span`: `hook_span`

Do NOT emit `LoweringError::ConflictingComponentMethod` — that is deferred to the type checker per CONTEXT.md decision.

Purpose: Close the three verification gaps from 05-VERIFICATION.md. All three share a single root cause: `entity_property` in `writ-parser/src/parser.rs` line 2583 uses `just(Token::Comma)` which requires a trailing comma after every entity property. When the last member of an entity is a property (no trailing comma before `}`), the parser fails to match `entity_property`, the entire `entity_decl` fails, and the input falls through to expression-statement error recovery. In `writ-parser/src/parser.rs`, line 2583, change:

```rust
.then_ignore(just(Token::Comma))
```

to:

```rust
.then_ignore(just(Token::Comma).or_not())
```

This makes the trailing comma optional on entity properties, matching the pattern already used by `entity_use` at line 2545. The `entity_member` choice parser (line 2588) tries `entity_use`, `entity_on`, `entity_fn`, then `entity_property` as a catch-all. **Do NOT change any other parser code.** The fix is exactly one token change: `just(Token::Comma)` to `just(Token::Comma).or_not()`.

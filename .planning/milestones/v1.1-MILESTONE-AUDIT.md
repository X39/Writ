### Issues Found

**1. BROKEN: Hex/Binary Literal Round-Trip (PARSE-02)**

`writ-compiler/src/lower/expr.rs:31` — `s.parse::<i64>().unwrap_or(0)` silently maps `0xFF` → 0 and `0b1010` → 0 since Rust's `str::parse::<i64>()` does not handle `0x`/`0b` prefixes. The snapshot `lowering_tests__lower_hex_binary_literals.snap` was accepted with `value: 0` for both.

**Fix needed:** Use `i64::from_str_radix()` with prefix detection, or store raw string in `AstExpr::IntLit`.

**2. ORPHANED: AstExpr::StructLit dead variant**

`writ-compiler/src/ast/expr.rs:137-144` — Defined but never constructed. Pre-Phase-13 residue.

**3. PARTIAL: String utilities not wired to lowering (LEX-03/LEX-05)**

`dedent_raw_string` and `process_escapes` exported from `writ_parser` but only consumed by tests. Lowering does `s.to_string()` for string literals without applying these transformations. May be intentional deferral to semantic pass.

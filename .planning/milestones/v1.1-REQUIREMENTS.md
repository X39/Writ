# Requirements: Writ Compiler

**Defined:** 2026-03-01
**Core Value:** Correct, spec-compliant desugaring of all Writ constructs while preserving source span information

## v1.1 Requirements

Requirements for spec v0.4 conformance update. Each maps to roadmap phases.

### Parser — Construction & Literals

- [x] **PARSE-01**: Parser accepts `new Type { field: value }` construction syntax with `KwNew` token and `Expr::New` CST node — Phase 10
- [x] **PARSE-02**: Parser matches hex (`0xFF`) and binary (`0b1010`) literal tokens in expression atoms — Phase 10 (partial: lowering bug, fixed v1.2 Phase 14)

### Parser — Type System & Paths

- [x] **TYPE-01**: Qualified type paths (`a::b::Type`) supported in `TypeExpr` (multi-segment) — Phase 9
- [x] **TYPE-02**: Root `::` prefix on paths preserved in CST (`Expr::Path` carries `rooted` flag) — Phase 9
- [x] **TYPE-03**: `ImplDecl` supports generic parameters (`impl<T> Contract<T> for Type<T>`) — Phase 11

### Parser — Declarations

- [x] **DECL-01**: Lifecycle hooks (`on create`, `on finalize`, `on serialize`, `on deserialize`) parsed inside struct bodies with CST representation — Phase 10
- [x] **DECL-02**: `self` and `mut self` valid as first parameter in function/method declarations with CST representation — Phase 10
- [x] **DECL-03**: Bodyless operator signatures (`operator +(other: T) -> R;`) parsed inside contract bodies — Phase 11
- [x] **DECL-04**: `DlgDecl` carries `attrs` and `vis` fields in CST; visibility not silently dropped — Phase 9
- [x] **DECL-05**: Non-extern `component` declarations produce a parser error — Phase 11
- [x] **DECL-06**: `extern fn` accepts dotted qualified names (`extern fn Entity.getOrCreate<T>() -> T;`) — Phase 11
- [x] **DECL-07**: `extern` declarations carry visibility modifiers (`pub extern fn ...`) — Phase 11

### Parser — Operators & Expressions

- [x] **EXPR-01**: Bit-shift operators `<<`/`>>` lexed, parsed, and added to `BinaryOp` enum — Phase 10
- [x] **EXPR-02**: `BitAnd` and `BitOr` added to `OpSymbol` enum and operator overloading parser/lowering — Phase 10
- [x] **EXPR-03**: `^` prefix expression validated as only valid inside bracket access context — Phase 11
- [x] **EXPR-04**: `spawn detached expr` parses as a single construct with dedicated CST node (not nested Spawn/Detached) — Phase 11
- [x] **EXPR-05**: `defer` only accepts a block expression, not arbitrary expressions — Phase 11

### Lexer

- [x] **LEX-01**: Raw strings enforce opening `"""` must be followed by a newline
- [x] **LEX-02**: Raw strings enforce closing `"""` must appear on its own line
- [x] **LEX-03**: Raw string leading whitespace dedentation applied (common prefix stripped)
- [x] **LEX-04**: `\u{XXXX}` unicode escapes in formattable strings correctly handled (not treated as `{XXXX}` interpolation)
- [x] **LEX-05**: Invalid escape sequences (`\q`, `\p`, etc.) rejected with lexer error

### Lowering — Dialogue & Localization

- [x] **DLG-01**: Namespace threaded to FNV-1a localization key generation (not always empty string) — Phase 12
- [x] **DLG-02**: Interpolation slot identities preserved in localization content (`{name}` not replaced with `{expr}`) — Phase 12
- [x] **DLG-03**: Choice label localization keys emitted in lowered output (not discarded with `let _ = key`) — Phase 12
- [x] **DLG-04**: `say()` emitted for non-localized context; `say_localized()` only when `#key` is present or auto-generated — Phase 12
- [x] **DLG-05**: Speaker scope saved/restored across `$ if` and `$ match` dialogue branches — Phase 12

### Lowering — Entity

- [x] **ENT-01**: Entity lowering recognizes all six lifecycle hooks: `create`, `destroy`, `interact`, `finalize`, `serialize`, `deserialize` — Phase 13
- [x] **ENT-02**: Entity components modeled as host-managed component slots (not lowered as inline struct fields) — Phase 13
- [x] **ENT-03**: Implicit `self`/`mut self` injected into operator methods and lifecycle hook method signatures — Phase 13
- [x] **ENT-04**: Entity lowering produces distinct `AstDecl::Entity` variant (not reusing `AstDecl::Struct`) — Phase 13

### Lowering — Misc

- [x] **MISC-01**: `IndexSet` contract name fixed from `"IndexMut"` to `"IndexSet"` in `op_symbol_to_contract` — Phase 13
- [x] **MISC-02**: Attribute named argument separator changed from `:` to `=` in parser — Phase 11
- [x] **MISC-03**: `Stmt::DlgDecl` dead code variant removed from CST — Phase 9

## Future Requirements

### Type Checking & Semantic Analysis (v2.0+)

- **SEM-01**: Exhaustiveness checking on `match` arms
- **SEM-02**: Scope/shadowing/forward-reference analysis
- **SEM-03**: Closure capture semantics enforcement (`let` by value, `let mut` by reference)
- **SEM-04**: Name conflict detection at `using` usage sites
- **SEM-05**: `[Conditional]` semantic validation (fn-only, must have fallback)
- **SEM-06**: `[Singleton]` semantic validation (entity-only, at-most-one-instance)
- **SEM-07**: `?` / `!` / `try` type checking and desugaring
- **SEM-08**: Namespace/`using` placement rule enforcement

### Code Generation (v2.0+)

- **CODEGEN-01**: `Entity.destroy`/`isAlive` → dedicated IL instructions
- **CODEGEN-02**: `entity[Component]` → `GET_COMPONENT` IL dispatch
- **CODEGEN-03**: Architecture-specific `[Import]` library/symbol resolution

## Out of Scope

| Feature | Reason |
|---------|--------|
| Type checking / name resolution | Separate compiler phase after lowering |
| Code generation (LLVM, WASM, bytecode) | Downstream of AST |
| Runtime implementation | Separate crate (`writ-runtime`) |
| Macro system | No macros in current spec |
| Optimization passes | Premature at this stage |
| `writ loc export/check/import` toolchain | CLI tooling, not compiler core |

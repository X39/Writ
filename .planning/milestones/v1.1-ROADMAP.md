## Phase Details

### Phase 8: Lexer Fixes
**Goal**: The lexer correctly validates raw string delimiters, strips leading whitespace dedentation, handles unicode escapes inside formattable strings, and rejects invalid escape sequences
**Depends on**: Nothing (foundational — lexer is the first pass)
**Requirements**: LEX-01, LEX-02, LEX-03, LEX-04, LEX-05
**Success Criteria** (what must be TRUE):
  1. A raw string whose opening `"""` is not followed by a newline produces a lexer error
  2. A raw string whose closing `"""` is not on its own line produces a lexer error
  3. Raw string content has its common leading whitespace prefix stripped before the token value is produced
  4. `\u{XXXX}` inside a formattable string is treated as a unicode escape, not as an interpolation slot `{XXXX}`
  5. An invalid escape sequence such as `\q` or `\p` produces a lexer error rather than silently passing through
**Plans**: 2 plans
  - [x] 08-01-PLAN.md — Raw string delimiter validation and unicode escape handling (LEX-01, LEX-02, LEX-04)
  - [x] 08-02-PLAN.md — Dedentation utility and escape sequence validation (LEX-03, LEX-05)

### Phase 9: CST Type System Additions
**Goal**: The CST type definitions carry all fields required by spec v0.4 — multi-segment qualified paths, a rooted-path flag, attrs/vis on DlgDecl — and the dead `Stmt::DlgDecl` variant is removed
**Depends on**: Phase 8
**Requirements**: TYPE-01, TYPE-02, DECL-04, MISC-03
**Success Criteria** (what must be TRUE):
  1. `TypeExpr` accepts multi-segment qualified paths (`a::b::Type`) and produces a CST node with all segments preserved
  2. `Expr::Path` carries a `rooted` boolean field that is set when the path begins with `::`
  3. `DlgDecl` in the CST has `attrs` and `vis` fields populated from the source — neither is silently dropped
  4. `Stmt::DlgDecl` no longer exists in the CST type definitions; the codebase compiles without it
**Plans**: 2 plans
  - [x] 09-01-PLAN.md — TypeExpr::Qualified and Expr::Path rooted flag (TYPE-01, TYPE-02)
  - [x] 09-02-PLAN.md — DlgDecl attrs/vis and Stmt::DlgDecl removal (DECL-04, MISC-03)

### Phase 10: Parser — Core Syntax
**Goal**: The parser accepts `new` keyword construction, hex and binary integer literals, struct lifecycle hooks, self/mut-self parameters, and the bit-shift and bitwise operators
**Depends on**: Phase 9
**Requirements**: PARSE-01, PARSE-02, DECL-01, DECL-02, EXPR-01, EXPR-02
**Success Criteria** (what must be TRUE):
  1. `new Type { field: value }` parses successfully and produces an `Expr::New` CST node; the old brace-construction syntax is rejected
  2. `0xFF` and `0b1010` parse as integer literal atoms in expressions
  3. `on create`, `on finalize`, `on serialize`, and `on deserialize` parse inside struct bodies and produce lifecycle hook CST nodes
  4. `self` and `mut self` are accepted as the first parameter in a function or method declaration and produce a distinct CST parameter variant
  5. `<<` and `>>` lex and parse as binary operators and appear in the `BinaryOp` enum
  6. `BitAnd` and `BitOr` (`&` and `|`) are added to the `OpSymbol` enum and are accepted by the operator overloading parser
**Plans**: 2 plans
  - [x] 10-01-PLAN.md — CST + Lexer + Parser + Lowering for all 6 features (PARSE-01, PARSE-02, DECL-01, DECL-02, EXPR-01, EXPR-02)
  - [x] 10-02-PLAN.md — Comprehensive tests for all 6 features (PARSE-01, PARSE-02, DECL-01, DECL-02, EXPR-01, EXPR-02)

### Phase 11: Parser — Declarations and Expressions
**Goal**: The parser enforces all remaining v0.4 declaration and expression rules — impl generics, bodyless operator signatures, component-declaration errors, extern qualified names, extern visibility, contextual caret, spawn-detached, defer-block-only, and the attribute separator fix
**Depends on**: Phase 10
**Requirements**: TYPE-03, DECL-03, DECL-05, DECL-06, DECL-07, EXPR-03, EXPR-04, EXPR-05, MISC-02
**Success Criteria** (what must be TRUE):
  1. `impl<T> Contract<T> for Type<T>` parses successfully; the generic parameter list is preserved in the `ImplDecl` CST node
  2. `operator +(other: T) -> R;` (bodyless) parses inside a contract body and produces an operator signature CST node
  3. A non-`extern component` declaration produces a parser error rather than parsing silently
  4. `extern fn Entity.getOrCreate<T>() -> T;` with a dotted qualified name parses successfully
  5. `pub extern fn` parses; visibility modifiers are preserved on extern declarations
  6. `^` as a prefix expression produces a parse error except inside a bracket-access context
  7. `spawn detached expr` parses as a single construct with a dedicated CST node (not nested Spawn + Detached)
  8. `defer { ... }` accepts only a block; `defer expr` produces a parse error
  9. Attribute named arguments use `=` as separator (`[Attr(key = value)]`); `:` is rejected
**Plans**: 2 plans
  - [x] 11-01-PLAN.md — CST/AST/Parser/Lowering implementation for all 9 requirements (TYPE-03, DECL-03, DECL-05, DECL-06, DECL-07, EXPR-03, EXPR-04, EXPR-05, MISC-02)
  - [x] 11-02-PLAN.md — Comprehensive tests for all 9 requirements (TYPE-03, DECL-03, DECL-05, DECL-06, DECL-07, EXPR-03, EXPR-04, EXPR-05, MISC-02)

### Phase 12: Lowering — Dialogue and Localization
**Goal**: Dialogue lowering produces correct localization keys (with namespace prefix, preserved slot identifiers, emitted choice keys), uses the right call (`say` vs `say_localized`), and correctly isolates speaker scope across branching dialogue constructs
**Depends on**: Phase 9
**Requirements**: DLG-01, DLG-02, DLG-03, DLG-04, DLG-05
**Success Criteria** (what must be TRUE):
  1. FNV-1a localization keys include the file namespace as a prefix (e.g., `my_module.fnv_key`) rather than `fnv_key` alone
  2. Interpolation slot names in localization content strings are preserved as-written (`{name}`) and are not replaced with a generic `{expr}` placeholder
  3. Choice label localization keys appear as named bindings in the lowered output rather than being discarded via `let _ = key`
  4. `say()` is emitted for dialogue lines with no localization key; `say_localized()` is emitted only when a `#key` override or auto-generated key is present
  5. After a `$ if` or `$ match` dialogue branch, the speaker reverts to its pre-branch value — a speaker set inside one branch does not leak into sibling branches or subsequent lines
**Plans**: 2 plans
  - [x] 12-01-PLAN.md — Implementation of all 5 dialogue lowering fixes (DLG-01, DLG-02, DLG-03, DLG-04, DLG-05)
  - [x] 12-02-PLAN.md — Comprehensive tests for all 5 dialogue lowering fixes (DLG-01, DLG-02, DLG-03, DLG-04, DLG-05)

### Phase 13: Lowering — Entity Model and Misc
**Goal**: Entity lowering handles all six lifecycle hooks, models components as host-managed slots (not inline struct fields), injects implicit self into operator and hook methods, emits a distinct `AstDecl::Entity` variant, and the `IndexSet` contract name is corrected
**Depends on**: Phase 10, Phase 12
**Requirements**: ENT-01, ENT-02, ENT-03, ENT-04, MISC-01
**Success Criteria** (what must be TRUE):
  1. Entity lowering produces hook registrations for all six hooks: `create`, `destroy`, `interact`, `finalize`, `serialize`, and `deserialize` — no hook is silently skipped
  2. Entity component declarations lower to host-managed component slot descriptors rather than inline struct field declarations
  3. Operator methods and lifecycle hook methods on entities receive an implicit `self` or `mut self` parameter injected by the lowering pass
  4. The lowered output for an entity declaration uses `AstDecl::Entity` rather than reusing `AstDecl::Struct`
  5. The `op_symbol_to_contract` mapping emits `"IndexSet"` for index-assignment operations, not `"IndexMut"`
**Plans**: 2 plans
  - [x] 13-01-PLAN.md — Implementation of entity model fixes, implicit self injection, and IndexSet correction (ENT-01, ENT-02, ENT-03, ENT-04, MISC-01)
  - [x] 13-02-PLAN.md — Comprehensive tests for all 5 entity and operator lowering fixes (ENT-01, ENT-02, ENT-03, ENT-04, MISC-01)

### Phase 14: Fix Hex/Binary Literal Lowering

**Requirements**: PARSE-02
**Gap Closure**: Closes gaps from v1.1 audit — PARSE-02 partial satisfaction, integration break, and broken hex/binary round-trip flow
**Success Criteria** (what must be TRUE):
  1. `0xFF` lowers to integer value `255`, not `0`
  2. `0b1010` lowers to integer value `10`, not `0`
  3. Decimal literals (`42`) continue to lower correctly
  4. Existing snapshot tests are updated to reflect correct values

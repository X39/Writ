## Phase Details

### Phase 14: Fix Hex/Binary Literal Lowering
**Goal**: The lowering pass correctly converts hex (`0xFF`) and binary (`0b1010`) integer literal strings to their numeric values using radix-aware parsing
**Depends on**: Phase 10
**Requirements**: PARSE-02
**Gap Closure**: Closes gaps from v1.1 audit — PARSE-02 partial satisfaction, integration break, and broken hex/binary round-trip flow
**Success Criteria** (what must be TRUE):
  1. `0xFF` lowers to integer value `255`, not `0`
  2. `0b1010` lowers to integer value `10`, not `0`
  3. Decimal literals (`42`) continue to lower correctly
  4. Existing snapshot tests are updated to reflect correct values
**Plans**: 1 plan
- [x] 14-01-PLAN.md — Fix radix-aware integer parsing, update snapshots, add edge-case tests (PARSE-02)

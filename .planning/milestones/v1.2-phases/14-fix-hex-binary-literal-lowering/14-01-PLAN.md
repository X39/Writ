<objective>
Fix the lowering pass so hex (`0xFF`) and binary (`0b1010`) integer literal strings are converted to their correct numeric values using radix-aware parsing instead of falling back to 0.

Purpose: The parser correctly lexes hex/binary tokens and collapses them to `Expr::IntLit(raw_text)`, but the lowering pass uses `str::parse::<i64>()` which only handles decimal. This causes all hex/binary literals to silently lower to `value: 0`, breaking the PARSE-02 round-trip flow.

Output: Corrected `lower_expr` with radix-aware parsing, updated snapshot, and edge-case test coverage.
</objective>

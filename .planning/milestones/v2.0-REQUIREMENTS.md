# Requirements: Writ IL Runtime

**Defined:** 2026-03-01
**Core Value:** Spec-compliant IL runtime as a reference implementation to validate future AST-to-IL codegen

## v2.0 Requirements

Requirements for the Writ IL runtime milestone. Each maps to roadmap phases.

### Module Format

- [x] **MOD-01**: Module reader can parse a spec-compliant binary module (200-byte header, 21 metadata tables, string/blob heaps, method bodies)
- [x] **MOD-02**: Module writer can produce a spec-compliant binary module from in-memory representation
- [x] **MOD-03**: ModuleBuilder API can programmatically construct valid IL modules for test authoring
- [x] **MOD-04**: Instruction enum represents all 98 opcodes with encode/decode round-trip correctness
- [x] **MOD-05**: MetadataToken newtype enforces 1-based indexing (0 = null token)
- [x] **MOD-06**: Module reader/writer round-trip produces identical output (write -> read -> write = identical bytes)

### VM Core

- [x] **VM-01**: Dispatch loop executes all 91 instructions via match-dispatch
- [x] **VM-02**: Typed register file stores values (Void/Int/Float/Bool/Ref/Entity) per-frame
- [x] **VM-03**: Managed call frame stack supports CALL/RET with register window sliding
- [x] **VM-04**: Arithmetic, comparison, and logic instructions produce spec-correct results
- [x] **VM-05**: Control flow instructions (BR, BR_TRUE, BR_FALSE, SWITCH) branch correctly
- [x] **VM-06**: String and type conversion instructions operate correctly
- [x] **VM-07**: Object model instructions (NEW, GET_FIELD, SET_FIELD, CAST, IS) work per spec

### Task Execution

- [x] **TASK-01**: Task scheduler manages tasks through 5-state lifecycle (Ready/Running/Suspended/Completed/Cancelled)
- [x] **TASK-02**: Suspend-and-confirm protocol correctly suspends at all 9 transition points
- [x] **TASK-03**: Defer stack executes in LIFO order on RET, crash, and CANCEL
- [x] **TASK-04**: Crash propagation unwinds full call stack, executing all defer handlers
- [x] **TASK-05**: Secondary crashes in defer handlers are logged and swallowed
- [x] **TASK-06**: ATOMIC_BEGIN/END prevents task interleaving and suppresses execution limits
- [x] **TASK-07**: SPAWN/JOIN/CANCEL concurrency instructions manage child tasks correctly
- [x] **TASK-08**: RuntimeHost trait provides synchronous interface for host communication

### Entity System

- [ ] **ENT-01**: Entity registry uses generation-indexed handles for stale-handle detection
- [ ] **ENT-02**: SPAWN_ENTITY/INIT_ENTITY construction buffering works per spec (batch flush on INIT)
- [ ] **ENT-03**: on_create lifecycle hook fires after INIT_ENTITY completes
- [ ] **ENT-04**: on_destroy lifecycle hook fires before host entity removal
- [ ] **ENT-05**: on_interact(who) lifecycle hook dispatches as a new task
- [ ] **ENT-06**: Singleton entities support GET_OR_CREATE semantics
- [ ] **ENT-07**: ENTITY_IS_ALIVE correctly distinguishes alive/destroyed/collected states
- [ ] **ENT-08**: Component field GET/SET suspends for host confirmation per suspend-and-confirm model

### Garbage Collection

- [ ] **GC-01**: Precise mark-and-sweep GC traces from register roots using TypeRef metadata
- [ ] **GC-02**: GC heap allocates and manages reference-type objects
- [ ] **GC-03**: Finalization queue populated during tracing; on_finalize hooks fire as scheduled tasks
- [ ] **GC-04**: GC runs at safe points (between ticks when all tasks are quiescent)
- [ ] **GC-05**: trait GcHeap abstraction allows replacing GC implementation without touching VM core

### Contract Dispatch

- [x] **DISP-01**: CALL_VIRT dispatches through contract table built from ImplDef rows at load time
- [x] **DISP-02**: writ-runtime virtual module provides Option, Result, Range types
- [x] **DISP-03**: writ-runtime virtual module provides all 17 contract definitions
- [x] **DISP-04**: writ-runtime virtual module provides primitive pseudo-TypeDefs (Int/Float/Bool/String)
- [x] **DISP-05**: writ-runtime virtual module provides Array<T> methods and Entity base type
- [x] **DISP-06**: Cross-module name resolution resolves TypeRef/MethodRef/FieldRef across modules

### Text Assembler

- [x] **ASM-01**: Human-readable text IL format with directives for modules, types, fields, methods
- [x] **ASM-02**: Two-pass assembler resolves forward label references correctly
- [x] **ASM-03**: Assembler produces spec-valid binary modules from text IL input
- [x] **ASM-04**: Round-trip test validates text IL -> binary -> disassemble -> compare

### Tooling

- [x] **TOOL-01**: Disassembler converts binary modules to human-readable text IL
- [x] **TOOL-02**: Standalone runner CLI loads and executes IL modules
- [x] **TOOL-03**: NullHost outputs say() to stdout, choice() returns 0, externs return defaults
- [x] **TOOL-04**: Runner CLI provides assemble/disasm/run subcommands

## Future Requirements

Deferred to v2.x or later. Tracked but not in current roadmap.

### Validation and Debug

- **VAL-01**: Module validation pass checks well-formedness before execution
- **VAL-02**: Debug info section (DebugLocal + SourceSpan) enables source-mapped errors
- **DBG-01**: Debug Adapter Protocol support for IDE debugging

### Execution Limits

- **EXEC-01**: Per-task instruction budget with configurable limit
- **EXEC-02**: Warning log when task approaches budget

### Persistence

- **SAVE-01**: Save/load serialization snapshots full VM state
- **SAVE-02**: IL coexistence on version mismatch (old stacks drain on old IL)
- **SAVE-03**: on_serialize/on_deserialize lifecycle hooks

### Advanced GC

- **GC-06**: Generational or incremental GC for reduced pause times
- **GC-07**: Cycle collection for reference-counted objects

### Runtime Extensions

- **EXT-01**: Hot reload command for updated modules
- **EXT-02**: writ-std standard library module (List, Map, etc.)
- **EXT-03**: Polymorphic inline caching at CALL_VIRT sites
- **EXT-04**: JIT compilation (separate writ-jit crate)

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| AST-to-IL codegen | Separate milestone; runtime is the reference implementation to test codegen against |
| JIT compilation | Reference interpreter must be complete first; separate crate/milestone |
| async/await for tasks | Spec uses cooperative yielding with explicit managed call stack; Rust async futures cannot be inspected or serialized |
| Script-defined components | Spec says components are extern-only, data-only |
| Exception tables | Spec uses crash propagation with defer unwinding, not structured exceptions |
| Name resolution / type checking | Separate compiler phase, lives in writ-compiler |
| Optimization passes | Premature; focus is correctness |

    **Create `src/loader.rs`** — LoadedModule and decode_and_reindex.

    ```rust
    use writ_module::{Instruction, Module};
    use crate::error::RuntimeError;

    pub struct LoadedModule {
        pub module: Module,
        pub decoded_bodies: Vec<Vec<Instruction>>,
    }

    impl LoadedModule {
        pub fn from_module(module: Module) -> Result<Self, RuntimeError> {
            let mut decoded_bodies = Vec::with_capacity(module.method_bodies.len());
            for (method_idx, body) in module.method_bodies.iter().enumerate() {
                let instructions = decode_and_reindex(&body.code, method_idx)?;
                decoded_bodies.push(instructions);
            }
            Ok(Self { module, decoded_bodies })
        }
    }
    ```

    **The `decode_and_reindex` function:**

    1. **Pass 1: Decode all instructions from raw bytes.** Use `Instruction::decode()` from writ-module. Track byte offset of each instruction start in a parallel vec: `byte_offsets: Vec<u32>`.

    2. **Build offset map:** Create a `HashMap<u32, usize>` mapping each `byte_offset -> instruction_index`. Also include the byte position immediately after the last instruction (for forward jumps to the end).

    3. **Pass 2: Rewrite branch targets.** For each instruction:
       - `Br { offset }`: The byte offset is relative to the CURRENT instruction's byte position. Compute `target_byte = current_byte_offset + offset`. Look up `offset_map[target_byte]` to get the instruction index. Replace `offset` with the instruction index cast to i32.
       - `BrTrue { r_cond, offset }` and `BrFalse { r_cond, offset }`: Same conversion.
       - `Switch { r_tag, offsets }`: Each offset is converted the same way.
       - `DeferPush { r_dst, method_idx }`: Per §3.11 and research open question #1, the `method_idx` field is actually a byte offset within the current method's code pointing to the handler block. Convert to instruction index using the same offset_map. The byte offset here is absolute (from method start), not relative.

    **IMPORTANT implementation notes:**
    - Branch offsets in the spec are RELATIVE to the current instruction's byte position (the start of the branch instruction). So `target_byte = branch_instr_byte_offset + branch_offset_value`.
    - DeferPush's field is an ABSOLUTE byte offset from method start (per §3.11 layout).
    - If a computed target_byte is not in the offset_map, return RuntimeError::DecodeError — this indicates a malformed module.
    - After rewriting, the offset/method_idx fields in the Instruction variants now hold instruction indices, not byte offsets. This is an intentional semantic change — the dispatch loop in Plan 02 will use these as instruction indices directly.

    **Create `tests/loader_tests.rs`** — integration tests for module loading.
    Use `writ_module::ModuleBuilder` to construct test modules programmatically.

    Test cases:
    1. **Simple linear body:** LoadInt + Ret → decoded as-is, no branch rewriting needed
    2. **Forward branch:** LoadInt + Br(forward) + LoadInt + Ret → Br target points to correct instruction index
    3. **Backward branch:** LoadInt + Br(-N) → Br target points to instruction 0
    4. **Conditional branch:** BrTrue with forward offset → target is correct instruction index
    5. **DeferPush reindexing:** DeferPush with byte offset → converted to instruction index

  <action>
    This task implements the three core modules. The dispatch loop is the largest piece — 98 match arms. Most arms are 1-3 lines (mechanical register operations). Complex arms delegate to helper functions.

    **Step 1: Create `src/dispatch.rs`** — the instruction dispatch function.

    The main function signature:
    ```rust
    pub(crate) enum ExecutionResult {
        Continue,               // instruction executed, keep going
        Completed(Value),       // task finished (RET from bottom frame)
        Suspended(RequestId),   // task suspended waiting for host
        Crash(String),          // task crashed
        LimitReached,           // execution budget exhausted
    }

    pub(crate) fn execute_one(
        task: &mut Task,
        module: &LoadedModule,
        heap: &mut BumpHeap,
        host: &mut dyn RuntimeHost,
        globals: &mut Vec<Value>,
        next_request_id: &mut u32,
    ) -> ExecutionResult { ... }
    ```

    Inside `execute_one`, get the current frame, fetch the instruction at `frame.pc`, increment `frame.pc`, then match:

    **Mechanical arms (1-3 lines each):**
    - `Nop` → no-op
    - `Crash { r_msg }` → read string from register (must be Ref to heap string), return `ExecutionResult::Crash(msg)`
    - `Mov { r_dst, r_src }` → `frame.registers[r_dst] = frame.registers[r_src]`
    - `LoadInt { r_dst, value }` → `frame.registers[r_dst] = Value::Int(value)`
    - `LoadFloat { r_dst, value }` → `frame.registers[r_dst] = Value::Float(value)`
    - `LoadTrue/LoadFalse/LoadNull` → set register to Bool(true)/Bool(false)/Ref(null) or Void
    - `LoadString { r_dst, string_idx }` → read string from module's string_heap using `writ_module::heap::read_string(&module.module.string_heap, string_idx)`, allocate as heap string, store Ref
    - Integer arithmetic (`AddI/SubI/MulI/DivI/ModI/NegI`): extract Int values, compute, store Int result. DivI/ModI: if divisor is 0, return Crash("division by zero")
    - Float arithmetic (`AddF/SubF/MulF/DivF/ModF/NegF`): extract Float values, compute, store Float result
    - Bitwise (`BitAnd/BitOr/Shl/Shr/Not`): extract Int values, compute, store Int
    - Comparison (`CmpEqI/CmpEqF/CmpEqB/CmpEqS/CmpLtI/CmpLtF`): compare values, store Bool. **CmpEqS: resolve BOTH HeapRefs to string content via heap.read_string() and compare strings by content, never by HeapRef index**
    - `Br { offset }` → `frame.pc = offset as usize` (offset is already an instruction index after reindexing)
    - `BrTrue { r_cond, offset }` → if register is Bool(true), set pc = offset as usize
    - `BrFalse { r_cond, offset }` → if register is Bool(false), set pc = offset as usize
    - `Switch { r_tag, offsets }` → get Int from register, use as index into offsets vec, set pc = offsets[idx] as usize. If index out of range, crash
    - `RetVoid` → same as Ret but with Value::Void
    - Conversion (`I2f/F2i/I2s/F2s/B2s`): I2f=Int→Float, F2i=Float→Int(truncate), I2s/F2s/B2s=convert to string and allocate on heap
    - `Convert { r_dst, r_src, target_type }` → placeholder: mov value (full conversion needs type system from Phase 19)
    - `StrConcat { r_dst, r_a, r_b }` → read both strings from heap, concatenate, alloc new string, store Ref
    - `StrBuild { r_dst, count, r_base }` → read `count` strings starting at r_base, concatenate all, alloc, store
    - `StrLen { r_dst, r_str }` → read string from heap, store Int(len)
    - `Box { r_dst, r_val }` → `heap.alloc_boxed(registers[r_val])`, store Ref
    - `Unbox { r_dst, r_boxed }` → get boxed value from heap, store in register
    - Option/Result/Enum type ops: use heap enum objects
      - `WrapSome` → alloc enum(tag=1, fields=[val])
      - `Unwrap` → extract field 0 from Some variant, crash if None
      - `IsSome/IsNone` → check tag, return Bool
      - `WrapOk/WrapErr` → alloc enum with tag 0/1
      - `UnwrapOk` → extract field 0, crash if Err
      - `IsOk/IsErr` → check tag
      - `ExtractErr` → extract error field
      - `NewEnum { r_dst, type_idx, tag, field_count, r_base }` → collect fields from registers, alloc enum
      - `GetTag` → get tag from enum heap object
      - `ExtractField` → get field from enum heap object
    - Array ops: use heap array objects
      - `NewArray` → alloc empty array
      - `ArrayInit` → alloc array with N elements from registers
      - `ArrayLoad/ArrayStore` → get/set array element by index
      - `ArrayLen` → return Int(length)
      - `ArrayAdd/ArrayRemove/ArrayInsert/ArraySlice` → mutate array

    **Complex arms (delegate to helpers):**
    - `Ret { r_src }` → call `execute_ret(task, frame.registers[r_src])`
      Helper: pops current frame. If call_stack empty → task completed. Else: write return value to caller frame's `registers[return_register]`. **Note: In this plan, defer handling during RET is stubbed (defers execute in Plan 03). Just pop the frame for now. Document with a TODO comment.**
    - `Call { r_dst, method_idx, r_base, argc }` → call `push_call_frame(task, method_idx, r_base, argc, r_dst, module)`
      Helper: create new CallFrame with registers sized from method body register_types, copy args from caller frame r_base..r_base+argc-1 to new frame r0..argc-1, push onto call_stack. Store r_dst as return_register on the new frame.
    - `TailCall { method_idx, r_base, argc }` → **Stub defer execution (Plan 03 adds real defer handling).** Copy args to temp buffer, replace current frame's method_idx/pc/registers, copy args from buffer to new registers r0..argc-1. Resets defer_stack to empty.
    - `CallExtern { r_dst, extern_idx, r_base, argc }` → collect args from registers, create HostRequest::ExternCall, call `suspend_for_host`. If host is NullHost (immediate response), place response in r_dst and continue. For general case: return `ExecutionResult::Suspended`.
      - Helper `suspend_for_host`: sets task.pending_request, task.state = Suspended, increments suspend_count, returns Suspended with RequestId
      - **Key: For NullHost (and any host that returns immediately from on_request), instead of suspending, directly place the response and continue. This means calling host.on_request immediately within the dispatch.** The NullHost auto-confirm behavior per CONTEXT.md means tasks never actually suspend in tests.
    - `CallVirt { .. }` → return `ExecutionResult::Crash("CALL_VIRT: contract dispatch not available".into())` per CONTEXT.md
    - `NewDelegate { r_dst, method_idx, r_target }` → alloc delegate on heap
    - `CallIndirect { r_dst, r_delegate, r_base, argc }` → get delegate from heap, extract method_idx, push call frame
    - Entity instructions (`SpawnEntity/InitEntity/DestroyEntity/GetComponent/GetOrCreate/FindAll/EntityIsAlive`) → dispatch as HostRequests. Since NullHost auto-confirms, these return defaults immediately in tests.
    - `LoadGlobal { r_dst, global_idx }` → read from globals vec. If atomic_depth > 0, track lock (stub for Plan 03 full implementation)
    - `StoreGlobal { global_idx, r_src }` → write to globals vec. Same atomic stub.
    - `AtomicBegin/AtomicEnd` → increment/decrement task.atomic_depth. Full locking logic in Plan 03.
    - `DeferPush { r_dst, method_idx }` → push method_idx (now instruction index) onto frame.defer_stack (actual execution in Plan 03)
    - `DeferPop` → pop from defer_stack (Plan 03 executes it)
    - `DeferEnd` → no-op marker in Plan 02 (signals end of defer handler in Plan 03)
    - `SpawnTask/SpawnDetached` → stub: return Crash("task spawning not yet implemented") — Plan 03 implements
    - `Join` → stub: suspend for host (NullHost confirms immediately)
    - `Cancel` → stub: return Crash("cancel not yet implemented") — Plan 03 implements

    **Step 2: Create `src/scheduler.rs`** — basic task scheduler.

    ```rust
    pub struct Scheduler {
        pub(crate) tasks: HashMap<TaskId, Task>,
        pub(crate) ready_queue: VecDeque<TaskId>,
        pub(crate) next_task_index: u32,
        pub(crate) globals: Vec<Value>,
        pub(crate) global_locks: HashMap<u32, TaskId>,
    }
    ```

    Methods:
    - `new()` → empty scheduler
    - `create_task(method_idx, args, parent_id) -> TaskId` → creates a new Task in Ready state, builds initial CallFrame from LoadedModule method body, copies args to registers, adds to ready_queue
    - `run_one_task(module, heap, host, limit) -> Option<ExecutionResult>` → takes next task from ready_queue, sets to Running, loops calling `execute_one` until ExecutionResult is not Continue, handles result (Completed/Suspended/Crash/LimitReached), returns result
    - `task_state(task_id) -> Option<TaskState>` → inspect task state

    ```rust
    pub struct RuntimeBuilder<H: RuntimeHost = NullHost> {

    }
    ```

    Methods on Runtime:
    - `tick(&mut self, delta_time: f64, limit: ExecutionLimit) -> TickResult` → iterate ready tasks, execute each within budget, return result
    - `confirm(&mut self, request_id: RequestId, response: HostResponse) -> Result<(), RuntimeError>` → find suspended task by request_id, validate response matches request kind, store response in frame's r_dst, set task to Ready
    - `spawn_task(&mut self, method_idx: usize, args: Vec<Value>) -> Result<TaskId, RuntimeError>` → create task via scheduler
    - `task_state(&self, task_id: TaskId) -> Option<TaskState>` → delegate to scheduler
    - `register_value(&self, task_id: TaskId, reg: u16) -> Option<Value>` → read register from task's top frame
    - `call_depth(&self, task_id: TaskId) -> Option<usize>` → return call_stack.len()

    **IMPORTANT implementation notes:**
    - The dispatch loop must handle ALL 98 Instruction variants. If any arm is missing, the match will be non-exhaustive and won't compile.
    - For host interaction: call `host.on_request(id, &req)` immediately. If response is not Error, use the response directly (e.g., place Value in r_dst). If Error, crash the task. This means NullHost auto-confirms without ever actually suspending the task — matching the CONTEXT.md decision.
    - For real hosts that need async confirmation: the dispatch returns Suspended, and Runtime::confirm() resumes later. But NullHost never hits this path.
    - Global storage: initialize from GlobalDefRow init_value blobs (or default to Void for each global). Size the globals vec to module.global_defs.len().
    - The scheduler lock strategy per CONTEXT.md: "simpler correct approach for Phase 17 is always hold the scheduler lock for the full run_task call." For Phase 17, do NOT use Mutex/locking. Keep everything single-threaded. The multi-thread-ready design (internal Mutex) can be added in a future phase. This avoids unnecessary complexity now.

    ```rust
    /// Helper: builds a module with one method containing the given instructions,

    fn run_simple(instructions: Vec<Instruction>, reg_count: u16) -> Runtime<NullHost> {
    ```

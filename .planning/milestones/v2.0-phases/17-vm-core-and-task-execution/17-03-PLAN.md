    **Part A: Defer execution engine**

    Add helper functions to scheduler.rs (or a new defer.rs module):

    ```rust
    /// Execute a defer handler at instruction index `handler_pc` within the current method.
    /// The handler is a block of instructions ending with DeferEnd.
    /// Returns Ok(()) on success or Err(crash_message) on secondary crash.
    pub(crate) fn execute_defer_handler(
        task: &mut Task,
        handler_pc: usize,
        module: &LoadedModule,
        heap: &mut BumpHeap,
        host: &mut dyn RuntimeHost,
        globals: &mut Vec<Value>,
        next_request_id: &mut u32,
    ) -> Result<(), String>
    ```

    Implementation: Save current frame.pc, set frame.pc = handler_pc. Execute instructions in a loop (calling execute_one) until DeferEnd is encountered. DeferEnd returns a special ExecutionResult variant (add `DeferComplete` to the enum if needed, or just break the loop when DeferEnd is hit). If a Crash occurs during defer execution, return Err(crash_message) — this is a secondary crash. Restore original frame.pc after handler completes.

    **Actually, a simpler approach:** The defer handler is a code region within the same method body. When a defer needs to execute:
    1. Set frame.pc to the handler's instruction index
    2. Execute instructions until DeferEnd instruction is hit
    3. DeferEnd causes the loop to break
    4. Control returns to whoever triggered the defer (RET handler or crash unwind)

    If a crash occurs during defer execution (e.g., the handler hits a CRASH instruction or division by zero), catch it, log as secondary crash, continue to next defer handler.

    **Update execute_ret in dispatch.rs:**
    Before popping the current frame, execute all defer handlers in LIFO order:
    ```rust
    fn execute_ret(task: &mut Task, ret_val: Value, module: ..., heap: ..., host: ..., globals: ..., next_req_id: ...) -> ExecutionResult {
        // Step 1: Run defers in LIFO order
        while let Some(handler_pc) = task.call_stack.last_mut().unwrap().defer_stack.pop() {
            if let Err(secondary_crash) = execute_defer_handler(task, handler_pc, module, heap, host, globals, next_req_id) {
                host.on_log(LogLevel::Error, &format!("secondary crash in defer: {}", secondary_crash));
            }
        }
        // Step 2: Pop frame
        let popped = task.call_stack.pop().unwrap();
        // Step 3: If stack empty -> completed; else deliver ret_val
        if task.call_stack.is_empty() {
            task.state = TaskState::Completed;
            task.return_value = Some(ret_val);
            ExecutionResult::Completed(ret_val)
        } else {
            let caller = task.call_stack.last_mut().unwrap();
            caller.registers[popped.return_register as usize] = ret_val;
            ExecutionResult::Continue
        }
    }
    ```

    Also update TailCall to execute defers before replacing frame.

    **Part B: Crash propagation (full call stack unwind)**

    Add `execute_crash` function:
    ```rust
    pub(crate) fn execute_crash(
        task: &mut Task, msg: String,
        module: &LoadedModule, heap: &mut BumpHeap, host: &mut dyn RuntimeHost,
        globals: &mut Vec<Value>, next_request_id: &mut u32,
    ) {
        // Build crash info (stack trace) BEFORE unwinding
        let crash_info = CrashInfo {
            message: msg.clone(),
            stack_trace: task.call_stack.iter().rev().map(|f| StackFrame {
                method_idx: f.method_idx,
                method_name: /* resolve from module string heap */ ,
                pc: f.pc,
            }).collect(),
        };

        // Unwind all frames, executing defers at each level
        while !task.call_stack.is_empty() {
            while let Some(handler_pc) = task.call_stack.last_mut().unwrap().defer_stack.pop() {
                if let Err(e) = execute_defer_handler(task, handler_pc, module, heap, host, globals, next_request_id) {
                    host.on_log(LogLevel::Error, &format!("secondary crash in defer during crash unwind: {}", e));
                    // TASK-05: swallow secondary crash, continue unwinding
                }
            }
            task.call_stack.pop();
        }

        task.state = TaskState::Cancelled;
        task.crash_info = Some(crash_info.clone());
        host.on_log(LogLevel::Error, &format!("task {} crashed: {}", task.id.index, crash_info.message));
    }
    ```

    Update all dispatch arms that currently return `ExecutionResult::Crash(msg)` to instead call `execute_crash` and return `ExecutionResult::Completed` (task is now terminal). Actually, the scheduler should handle the Crash result — when `execute_one` returns `Crash(msg)`, the scheduler calls `execute_crash` on the task.

    **Part D: Concurrency instructions**

    Replace stubs for SPAWN_TASK, SPAWN_DETACHED, JOIN, CANCEL with real implementations.

    `SpawnTask { r_dst, method_idx, r_base, argc }`:
    - Collect args from registers r_base..r_base+argc-1
    - Create new task via scheduler with parent_id = Some(current_task.id)
    - Add child to current task's scoped_children
    - Store child TaskId in r_dst as Value::Int(child.id.index as i64) or a new value representation. **Design decision:** TaskId is not directly representable as a Value variant. Since there's no Value::Task variant, store it as Value::Int(packed) where the int encodes the handle. OR, better: add the task_id to a task-local map and return an index. **Actually:** The spec says SPAWN_TASK returns a task handle in r_dst. The Value enum has no Task variant. The simplest approach: store the GenHandle's index+generation packed into an i64 (index in high 32 bits, generation in low 32 bits). JOIN/CANCEL unpack it. This is an internal representation detail — the runtime controls both sides.

    `SpawnDetached { r_dst, method_idx, r_base, argc }`:
    - Same as SpawnTask but with parent_id = None (root-level task)
    - Do NOT add to current task's scoped_children

    `Join { r_dst, r_task }`:
    - Unpack TaskId from register value
    - If target task is already Completed/Cancelled: place result/Void in r_dst, continue
    - If target task is still running/ready: suspend current task with HostRequest::Join. The scheduler handles this by checking the target task state and waking the joining task when target completes.
    - **Implementation:** Don't use host for JOIN — handle it entirely in the scheduler. When the scheduler runs tasks and a task completes, check if any other task is waiting to JOIN it, and move that task to Ready.

    `Cancel { r_task }`:
    - Unpack TaskId from register value
    - Call `cancel_task_tree(scheduler, target_task_id, module, heap, host, globals, next_req_id)`

    **Scoped task cancellation (recursive):**
    ```rust
    fn cancel_task_tree(scheduler, task_id, module, heap, host, globals, next_req_id) {
        // Cancel all scoped children FIRST (depth-first recursive)
        let children = scheduler.tasks[task_id].scoped_children.clone();
        for child_id in children {
            cancel_task_tree(scheduler, child_id, module, heap, host, globals, next_req_id);
        }
        // Then cancel this task (full crash unwind with defers)
        let task = scheduler.tasks.get_mut(&task_id).unwrap();
        if matches!(task.state, TaskState::Completed | TaskState::Cancelled) {
            return; // already terminal
        }
        execute_crash(task, "task cancelled".into(), module, heap, host, globals, next_req_id);
    }
    ```

    pub(crate) fn pack_task_id(id: TaskId) -> Value {
        Value::Int(((id.index as i64) << 32) | (id.generation as i64))
    }
    pub(crate) fn unpack_task_id(val: &Value) -> Option<TaskId> {
        if let Value::Int(packed) = val {
            let index = (*packed >> 32) as u32;
            let generation = (*packed & 0xFFFFFFFF) as u32;
            Some(TaskId::new(index, generation))
        } else { None }
    }

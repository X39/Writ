    **Step 3: Rewrite INIT_ENTITY handler.**

    ```rust
    Instruction::InitEntity { r_entity } => {
        let frame = task.call_stack.last().unwrap();
        let entity_id = extract_entity(&frame.registers[r_entity as usize]);

        // Commit initialization: flush buffered field writes
        let field_writes = match entity_registry.commit_init(entity_id) {
            Ok(writes) => writes,
            Err(e) => return ExecutionResult::Crash(format!("InitEntity: {}", e)),
        };

        // Apply buffered field writes to the heap object
        if let Ok(Some(data_ref)) = entity_registry.get_data_ref(entity_id) {
            for (field_idx, value) in field_writes {
                let _ = heap.set_field(data_ref, field_idx as usize, value);
            }
        }

        // Fire on_create hook if it exists (inline in same task)
        // Look up on_create method for this entity's type
        if let Ok(type_idx) = entity_registry.get_type_idx(entity_id) {
            if let Some(hook_method) = lookup_hook(module, type_idx, HookKind::OnCreate) {
                // Push a new call frame for on_create with self=entity_id
                let reg_count = module.module.method_bodies[hook_method].register_types.len();
                let mut hook_frame = CallFrame::new(hook_method, reg_count, r_entity);
                hook_frame.registers[0] = Value::Entity(entity_id);
                task.call_stack.push(hook_frame);
                return ExecutionResult::Continue;
            }
        }

        // Notify host
        let req_id = RequestId(*next_request_id);
        *next_request_id += 1;
        let req = HostRequest::InitEntity { task_id: task.id, entity: entity_id };
        let response = host.on_request(req_id, &req);
        match response {
            HostResponse::Confirmed | HostResponse::Value(_) => ExecutionResult::Continue,
            HostResponse::Error(e) => ExecutionResult::Crash(format!("InitEntity failed: {:?}", e)),
            _ => ExecutionResult::Continue,
        }
    }
    ```

    **Step 4: Rewrite DESTROY_ENTITY handler.**

    ```rust
    Instruction::DestroyEntity { r_entity } => {
        let frame = task.call_stack.last().unwrap();
        let entity_id = extract_entity(&frame.registers[r_entity as usize]);

        // Validate entity is alive (crash on stale or already-destroyed handle)
        if !entity_registry.is_alive(entity_id) {
            return ExecutionResult::Crash(format!(
                "DestroyEntity: entity {:?} is not alive (stale or already destroyed)",
                entity_id
            ));
        }

        // Fire on_destroy hook inline before marking destroyed
        if let Ok(type_idx) = entity_registry.get_type_idx(entity_id) {
            if let Some(hook_method) = lookup_hook(module, type_idx, HookKind::OnDestroy) {
                // Mark entity as destroyed FIRST to prevent on_destroy from re-destroying
                // Actually, per CONTEXT.md: on_destroy fires BEFORE host notification
                // and cascading destruction is synchronous. Fire hook inline.
                let reg_count = module.module.method_bodies[hook_method].register_types.len();
                let mut hook_frame = CallFrame::new(hook_method, reg_count, r_entity);
                hook_frame.registers[0] = Value::Entity(entity_id);
                task.call_stack.push(hook_frame);
                // After hook frame completes (RET), we still need to do the actual destroy.
                // Use a ExecutionResult variant or store pending destroy info.
                // Simpler approach: mark entity as Destroyed now, fire hook.
                // The hook runs with the entity already destroyed.
                // Actually, per CONTEXT decisions: on_destroy fires BEFORE host removal.
                // Let's destroy AFTER the hook returns.

                // For simplicity: store a "pending_destroy" on the task, then
                // after the hook frame returns, the scheduler checks for pending destroys.
                // OR: push a synthetic "do destroy" step after the hook frame.
                // Simplest: just mark destroyed now. on_destroy can still read entity state
                // since the heap data is still there. The entity handle becomes stale
                // for external users, but the inline on_destroy still has the data.

                // Decision: Destroy after hook. The on_destroy hook should see the entity as alive.
                // We'll use an ExecutionResult variant to signal "run hook then destroy".
            }
        }

        // Mark destroyed in registry
        if let Err(e) = entity_registry.destroy(entity_id) {
            return ExecutionResult::Crash(format!("DestroyEntity: {}", e));
        }

        // Notify host
        let req_id = RequestId(*next_request_id);
        *next_request_id += 1;
        let req = HostRequest::DestroyEntity { task_id: task.id, entity: entity_id };
        let response = host.on_request(req_id, &req);
        match response {
            HostResponse::Confirmed | HostResponse::Value(_) => ExecutionResult::Continue,
            HostResponse::Error(e) => ExecutionResult::Crash(format!("DestroyEntity failed: {:?}", e)),
            _ => ExecutionResult::Continue,
        }
    }
    ```

    NOTE on on_destroy ordering: The simplest correct approach is to fire on_destroy BEFORE marking the entity as destroyed. This way on_destroy sees the entity as alive. After on_destroy returns (RET pops the hook frame), the next instruction is a synthetic continuation that marks destroyed and notifies host. Implementation options:

    a) Use a two-phase approach: The DESTROY_ENTITY handler pushes the on_destroy hook frame AND decrements PC so that when the hook returns, DESTROY_ENTITY re-executes. Add a flag on the entity (e.g., state = Destroying) to skip the hook on re-execution.

    b) Simpler: Add an `EntityState::Destroying` state. First call sets Destroying and pushes hook. When hook returns and DESTROY_ENTITY re-fires, it sees Destroying state, skips hook, and completes destruction.

    Use approach (b). Add `Destroying` variant to EntityState. Update the destroy path accordingly.

Create the `writ-assembler` crate with lexer, AST, and recursive-descent parser for the `.writil` text IL format.

Purpose: Establishes the text-to-structure pipeline — the "read" side of the assembler. The parser produces an AST that the assembler (Plan 02) will consume to build binary modules.

Output: New `writ-assembler` crate that can tokenize and parse `.writil` text into a structured AST, with multi-error collection and line:column diagnostics.

    - Define `AssembleError` using thiserror with `#[error("error: {message} at line {line}, column {col}")]` format per the locked decision.
    - Include fields: `message: String`, `line: u32`, `col: u32`.
    - Add a convenience constructor: `AssembleError::new(message: impl Into<String>, line: u32, col: u32)`.

    **Step 3: Create lexer.rs.**
    - Define `TokenKind` enum with variants: `Directive` (`.module`, `.type`, `.field`, etc.), `Ident` (identifiers and mnemonic names), `IntLit(i64)`, `FloatLit(f64)`, `StringLit(String)`, `Register(u16)` (for `r0`, `r1`, etc.), `Label(String)` (for `.foo:` definitions), `LabelRef(String)` (for `.foo` references in instructions), `OpenBrace`, `CloseBrace`, `OpenParen`, `CloseParen`, `Comma`, `Colon`, `Arrow` (`->`), `DoubleColon` (`::`), `LAngle`, `RAngle`, `Newline`, `Eof`.
    - Define `Token` struct with `kind: TokenKind`, `line: u32`, `col: u32`.
    - Implement `pub fn tokenize(src: &str) -> Result<Vec<Token>, Vec<AssembleError>>`.
    - The tokenizer is line-oriented. Handle:
      - `//` comments (skip to end of line)
      - Directives: tokens starting with `.` followed by alphanumeric — if next non-whitespace char after the identifier is `:`, it is a `Label` definition (`.loop:`); otherwise it is a `Directive`.
      - Registers: `r` followed by digits -> `Register(n)`
      - String literals: `"..."` with basic escape sequences (`\\`, `\"`, `\n`, `\t`, `\0`)
      - Integer literals: decimal, `0x` hex, `0b` binary, with optional `_` separators
      - Float literals: digits with `.` and optional exponent
      - Symbols: `{`, `}`, `(`, `)`, `,`, `:`, `::`, `->`, `<`, `>`
      - Identifiers: alphanumeric + `_` + `.` (for qualified names like `MyNamespace.MyType`)
      - Case-insensitive instruction mnemonics are tokenized as `Ident` — the parser handles case-insensitive matching
    - Collect multiple errors (e.g., unterminated strings) and continue tokenizing.

    **Step 4: Create ast.rs.**
    - Define AST node types matching the text IL grammar from the RESEARCH.md:
      - `AsmModule { name: String, version: String, externs: Vec<AsmExtern>, types: Vec<AsmType>, contracts: Vec<AsmContract>, impls: Vec<AsmImpl>, globals: Vec<AsmGlobal>, extern_fns: Vec<AsmExternFn>, methods: Vec<AsmMethod> }` — top-level module
      - `AsmExtern { name: String, min_version: String }` — `.extern` directive
      - `AsmType { name: String, kind: AsmTypeKind, flags: u16, fields: Vec<AsmField> }` — `.type` block
      - `AsmTypeKind` — enum: `Struct`, `Enum`, `Entity`, `Component`
      - `AsmField { name: String, type_ref: AsmTypeRef, flags: u16 }` — `.field` inside `.type`
      - `AsmContract { name: String, methods: Vec<AsmContractMethod>, generic_params: Vec<String> }` — `.contract` block
      - `AsmContractMethod { name: String, signature: AsmMethodSig, slot: u16 }` — `.method` inside `.contract`
      - `AsmImpl { type_name: String, contract_name: String, methods: Vec<AsmMethod> }` — `.impl` block
      - `AsmMethod { name: String, params: Vec<AsmParam>, return_type: AsmTypeRef, registers: Vec<AsmRegDecl>, body: Vec<AsmStatement> }` — `.method` block
      - `AsmParam { name: String, type_ref: AsmTypeRef }` — method parameter
      - `AsmRegDecl { index: u16, type_ref: AsmTypeRef }` — `.reg` directive
      - `AsmStatement` — enum: `Label(String)`, `Instruction(AsmInstruction)`
      - `AsmInstruction { mnemonic: String, operands: Vec<AsmOperand>, line: u32, col: u32 }` — a single instruction
      - `AsmOperand` — enum: `Register(u16)`, `IntLit(i64)`, `FloatLit(f64)`, `StringLit(String)`, `LabelRef(String)`, `TypeRef(AsmTypeRef)`, `MethodRef(AsmMethodRef)`, `FieldRef(AsmFieldRef)`, `Token(u32)`
      - `AsmTypeRef` — enum: `Void`, `Int`, `Float`, `Bool`, `String_`, `Named(String)` (for `MyNamespace.MyType`), `Array(Box<AsmTypeRef>)`, `Generic(String, Vec<AsmTypeRef>)` (for `Option<MyType>`), `RawBlob(Vec<u8>)`
      - `AsmMethodRef { type_name: Option<String>, method_name: String, module_name: Option<String> }` — for `MyType::my_method` or `[OtherModule]Type::method`
      - `AsmFieldRef { type_name: String, field_name: String, module_name: Option<String> }` — for `MyType::field`
      - `AsmGlobal { name: String, type_ref: AsmTypeRef, flags: u16, init_value: Option<Vec<u8>> }` — `.global` directive
      - `AsmExternFn { name: String, signature: AsmMethodSig, import_name: String, flags: u16 }` — `.extern` function
      - `AsmMethodSig { params: Vec<AsmTypeRef>, return_type: AsmTypeRef }` — method signature for contract methods

  <action>
    **Implement parser.rs** as a hand-written recursive-descent parser.

    Define a `Parser` struct holding `tokens: &[Token]`, `pos: usize`, `errors: Vec<AssembleError>`.

    Core methods:
    - `peek() -> &Token` — look at current token
    - `advance() -> &Token` — consume and return current token
    - `expect(kind) -> Result<&Token, ()>` — consume if match, push error and return Err if not
    - `skip_newlines()` — consume all Newline tokens
    - `synchronize()` — on error, skip tokens until next `}`, `Newline` after a `Directive`, or `Eof` to enable error recovery

    Parse functions (each returns the parsed node or pushes errors and returns a default/skips):
    - `parse_module() -> AsmModule` — expects `.module` StringLit StringLit `{` items `}`
    - `parse_module_item()` — dispatches on directive: `.type` -> `parse_type()`, `.contract` -> `parse_contract()`, `.impl` -> `parse_impl()`, `.method` -> `parse_method()`, `.extern` -> `parse_extern()`, `.global` -> `parse_global()`
    - `parse_type() -> AsmType` — `.type` StringLit kind `{` fields `}`; kind is `struct`/`enum`/`entity`/`component` (case-insensitive)
    - `parse_field() -> AsmField` — `.field` StringLit type_ref flags_opt
    - `parse_contract() -> AsmContract` — `.contract` StringLit generic_params_opt `{` contract_methods `}`
    - `parse_contract_method() -> AsmContractMethod` — `.method` StringLit signature `slot` IntLit
    - `parse_impl() -> AsmImpl` — `.impl` type_name `:` contract_name `{` methods `}`
    - `parse_method() -> AsmMethod` — `.method` StringLit `(` params `)` `->` return_type `{` reg_decls instructions `}`
    - `parse_reg_decl() -> AsmRegDecl` — `.reg` Register type_ref (inside `.method` body, before instructions)
    - `parse_instruction() -> AsmInstruction` — mnemonic followed by comma-separated operands until Newline
    - `parse_operand() -> AsmOperand` — registers, int/float/string literals, label refs, type refs, method refs
    - `parse_type_ref() -> AsmTypeRef` — primitives (`int`, `float`, `bool`, `string`, `void`), named types, generics (`Array<int>`), `blob(...)` escape hatch

    **Key design decisions (per Claude's discretion):**
    - Use `{...}` blocks for all nesting (`.module { }`, `.type { }`, `.method { }`, `.contract { }`, `.impl { }`)
    - Register declarations appear at the top of `.method` blocks before any instructions, using `.reg r0 int` syntax (one per line)
    - Method signatures in text use `(param_type, param_type) -> return_type` shorthand for contract methods, and `(name type_ref, name type_ref) -> return_type` for method definitions with named params
    - Flags parsed as optional keywords: `pub` = 0x0001, `mut` = 0x0002, `static` = 0x0004 (or numeric literal for other flags)
    - Instructions are case-insensitive (compare mnemonic uppercased against spec names)

    **Update lib.rs** to wire lexer -> parser in the `assemble()` function (assembler step still stubbed).

    **Create tests/parse_tests.rs** with tests covering all behaviors listed above. Use `lexer::tokenize` + `parser::parse` to verify AST output. Test multi-error collection by parsing input with multiple deliberate errors.

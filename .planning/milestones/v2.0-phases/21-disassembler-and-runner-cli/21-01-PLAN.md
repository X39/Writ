<objective>
Implement the disassembler in `writ-assembler` that converts a binary `Module` struct into human-readable `.writil` text format, symmetric with the assembler.

Purpose: The disassembler is the inverse of the assembler, completing the text-to-binary-to-text round-trip. It enables developers to inspect binary modules and validates that the assembler produces correct output.

Output: `writ-assembler/src/disassembler.rs` with `disassemble()` and `disassemble_verbose()` public functions, plus unit and round-trip tests.
</objective>

  <action>
    Create `writ-assembler/src/disassembler.rs` implementing the binary-to-text disassembler:

    **Core structure:**
    - `pub fn disassemble(module: &Module) -> String` — clean output for editing
    - `pub fn disassemble_verbose(module: &Module) -> String` — adds hex offset and opcode byte comments
    - Internal `fn disassemble_inner(module: &Module, verbose: bool) -> String`

    **Helper function: `s(heap, offset)` -> resolves string heap offset to `&str`:**
    Use `writ_module::heap::read_string(&module.string_heap, offset).unwrap_or("")` throughout.

    **Type signature decoding:**
    - `fn decode_type_sig(blob_heap: &[u8], sig_offset: u32, module: &Module) -> String`
    - `fn decode_type_ref(blob: &[u8], pos: &mut usize, module: &Module) -> String`
    - Handles: 0x00=void, 0x01=int, 0x02=float, 0x03=bool, 0x04=string, 0x10+u32=named, 0x20=array
    - Named type tokens: extract the 1-based row index, subtract 1, look up `module.type_defs[idx].name` to resolve to a type name string. If token has table bits for TypeRef (check bits 31-24), look up type_refs instead.

    **Method signature decoding:**
    - `fn decode_method_sig(blob_heap: &[u8], sig_offset: u32, module: &Module) -> (Vec<String>, String)`
    - Reads u16(param_count), then return type blob, then param type blobs. Returns (param_types, return_type).

    **Disassembly output order (mirrors assembler's Phase 1 declaration order):**
    1. `.module "name" "version" {`
    2. `.extern "ModuleRefName" "min_version"` for each ModuleRefRow
    3. `.type "Name" kind {` for each TypeDefRow — emit `.field` directives for fields in range [field_list, next_type.field_list). Kind: 0=struct, 1=enum, 2=entity, 3=component. Field flags as `0xNNNN` hex.
    4. `.contract "Name" { .method "name" (params) -> ret slot N }` for each ContractDefRow — use method_list range to find contract methods. If contract has generic params (generic_list range), emit `<T, U>` after contract name.
    5. `.impl TypeName : ContractName { .method ... { body } }` for each ImplDefRow — resolve type_token and contract_token to names via type_defs/contract_defs tables. Use method_list range from ImplDef to find which MethodDef rows belong to this impl.
    6. `.global "name" type flags` for each GlobalDefRow
    7. `.extern fn "name" (sig) -> ret "import_name"` for each ExternDefRow
    8. Top-level `.method "name" (params) -> ret { body }` for each MethodDef NOT owned by a type or impl (unowned methods). Determine ownership by checking which methods fall in type_def.method_list ranges and impl_def.method_list ranges. Methods not in any range are top-level.
    9. `.export "name" kind token` for each ExportDefRow
    10. `.component_slot owner comp "name"` for each ComponentSlotRow (if any)
    11. `.locale "key" "locale" "value"` for each LocaleDefRow (if any)
    12. `.attribute target "name" blob` for each AttributeDefRow (if any)
    13. `}` closing the module

    **Method body disassembly:**
    - `fn disassemble_body(body: &MethodBody, module: &Module, verbose: bool, indent: &str) -> String`
    - Emit `.reg rN type` for each register_type entry (decode type blob from blob_heap)
    - Decode instructions from `body.code` using `Instruction::decode(&mut Cursor::new(&body.code))` in a loop
    - Map each Instruction variant to `(mnemonic, operands_vec)` via `fn instr_to_text(instr: &Instruction) -> (String, Vec<String>)`
    - Register operands: `rN`. Int operands: decimal. Float operands: use `{:.}` format (no trailing zeros). Token operands: emit raw u32 decimal value (assembler accepts integer tokens).
    - Branch offsets: emit raw i32 value (assembler accepts integer offsets directly)
    - For verbose mode: emit `// +0xNNNN` hex offset comment before each instruction
    - IMPORTANT: disassemble from `module.method_bodies[i].code` (raw bytes), NOT from LoadedModule (which reindexes branch offsets)

    **Instruction-to-mnemonic mapping (`fn instr_to_text`):**
    Match all 91 Instruction variants. The mnemonic is the SCREAMING_SNAKE_CASE name matching the assembler's map_instruction table exactly. Copy the pattern from the assembler's match arms (lines 501-728 of assembler.rs) but in reverse: Instruction variant -> (mnemonic_string, operand_strings).

    **Method ownership detection:**
    Build two sets at the start of disassembly:
    - `type_owned_methods: HashSet<usize>` — method indices in type_def method_list ranges
    - `impl_owned_methods: HashSet<usize>` — method indices in impl_def method_list ranges
    Top-level methods are those not in either set.

    **Wire into lib.rs:**
    Add `pub mod disassembler;` to `writ-assembler/src/lib.rs`.
    Add re-exports: `pub use disassembler::{disassemble, disassemble_verbose};`

    **Write tests in `disasm_basic.rs`:**
    - `disassemble_empty_module`: assemble minimal module, disassemble, check output contains `.module` and `}`
    - `disassemble_type_with_fields`: assemble a struct with 2 fields, disassemble, check `.type` and `.field` directives present
    - `disassemble_contract_with_methods`: assemble a contract, disassemble, check `.contract` and `.method slot` present
    - `disassemble_method_with_instructions`: assemble a method with LOAD_INT + RET, disassemble, check mnemonics present
    - `disassemble_impl_block`: assemble an impl block, disassemble, check `.impl TypeName : ContractName` present
    - `disassemble_verbose_includes_offsets`: use disassemble_verbose(), check `// +0x` comments appear

    **Write tests in `disasm_round_trip.rs`:**
    - `round_trip_minimal_module`: assemble -> disassemble -> reassemble, assert same table counts
    - `round_trip_type_with_fields`: struct with fields survives round-trip
    - `round_trip_method_with_body`: method with instructions survives round-trip
    - `round_trip_complex_module`: module with types, contracts, impls, methods, externs all survive round-trip
    - Each test: `let m1 = assemble(src).unwrap(); let text = disassemble(&m1); let m2 = assemble(&text).unwrap();` then assert `m1.type_defs.len() == m2.type_defs.len()`, etc.

    **Key pitfalls to avoid (from RESEARCH.md):**
    - String heap offset 0 is valid (empty string) — always use `read_string`, never assume offset 0 is an error
    - Method body index alignment: `method_bodies[i]` is parallel to `method_defs[i]` — assert this at start
    - Branch offsets are byte offsets in raw `Module.method_bodies[i].code` — NOT instruction indices
    - Token values in instructions are 1-based MetadataToken values — when resolving to names, subtract 1 for 0-based indexing and guard against 0 (null token)
    - MetadataToken layout: table_id in bits 31-24, 1-based row in bits 23-0. When resolving a token used as a type reference in an instruction operand, extract the row index with `token & 0x00FFFFFF` then subtract 1 for 0-based array access

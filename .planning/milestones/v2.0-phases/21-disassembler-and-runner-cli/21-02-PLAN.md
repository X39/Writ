    Constructor takes `(module: &Module, interactive: bool, verbose: bool)`. Builds `extern_names` by iterating `module.extern_defs` and resolving each `name` field via `read_string(&module.string_heap, ed.name)`.

    `impl RuntimeHost for CliHost`:
    - `on_request`: Match on `HostRequest` variants:
      - `ExternCall { extern_idx, args, .. }`:
        - Decode row index: `let row = (extern_idx & 0x00FF_FFFF) as usize; let idx = row.saturating_sub(1);`
        - Look up name: `self.extern_names.get(idx).map(|s| s.as_str()).unwrap_or("?")`
        - Match on name:
          - `"say"` -> Format arg: if `Value::Int(i)` print i, if `Value::Float(f)` print f, if `Value::Bool(b)` print b, if `Value::Ref(h)` print `<string@{h.0}>`. Print `[say] {text}`. Return `HostResponse::Value(Value::Void)`.
            **Known limitation (document in code comment):** When say() receives a string argument, the VM passes `Value::Ref(HeapRef)` because `LOAD_STRING` allocates on the GC heap. `CliHost.on_request()` has no access to the GC heap to dereference the HeapRef to actual string content. Primitive values (Int, Float, Bool) print correctly. Full string content printing requires either: (a) runtime-level pre-resolution of Ref args to string content before issuing HostRequest::ExternCall, or (b) passing a heap reference to the host. Both are deferred to a future phase. The end-to-end test validates the full say() stack path using LOAD_INT (which produces Value::Int, printable by CliHost).
          - `"choice"` -> If interactive, prompt on stdin and parse index. If non-interactive, print `[choice] auto-selecting 0`, return `HostResponse::Value(Value::Int(0))`.
          - Other -> print `[extern] {name}()`, return `HostResponse::Value(Value::Void)`.
      - `EntitySpawn { type_idx, .. }` -> print `[entity:spawn] type={type_idx}`, return `HostResponse::Confirmed`
      - `DestroyEntity { entity, .. }` -> print `[entity:destroy] entity={entity:?}`, return `HostResponse::Confirmed`
      - All others -> return `HostResponse::Confirmed` (FieldRead returns `Value::Int(0)`, GetComponent returns `Value::Void`)
    - `on_log`: Print `[{level:?}] {message}` to stderr.
    - `on_gc_complete`: If verbose, print `[gc] {stats:?}` to stderr.

    Add `pub fn print_stats(&self)` method: if verbose, print `[stats] instructions={}, tasks={}` to stderr.

    Increment `self.instructions_executed` — actually, CliHost doesn't observe individual instructions. Track `tasks_spawned` by counting EntitySpawn and ExternCall requests if verbose. Keep it simple: just count on_request calls.

    **Step 3: Create writ-cli/src/main.rs**

    Use clap derive macros:
    ```rust
    mod cli_host;

    use clap::{Parser, Subcommand};

    #[derive(Parser)]
    #[command(name = "writ", about = "Writ IL toolchain")]
    struct Cli {
        #[command(subcommand)]
        command: Commands,
    }

    #[derive(Subcommand)]
    enum Commands {
        /// Assemble .writil text to binary .writc
        Assemble {
            /// Input file (or '-' for stdin)
            input: String,
            /// Output file (default: input with .writc extension)
            #[arg(short, long)]
            output: Option<String>,
        },
        /// Disassemble binary .writc to .writil text
        Disasm {
            /// Input binary module file
            input: String,
            /// Add hex offsets and opcode byte comments
            #[arg(long)]
            verbose: bool,
        },
        /// Run a binary .writc module
        Run {
            /// Input binary module file
            input: String,
            /// Entry method name (default: "main")
            #[arg(long, default_value = "main")]
            entry: String,
            /// Interactive choice prompts
            #[arg(long)]
            interactive: bool,
            /// Print execution stats after run
            #[arg(long)]
            verbose: bool,
            // NOTE: `args: Vec<String>` for entry method parameter passing is DEFERRED to a future phase.
            // The locked decision specifies "if method accepts a parameter, pass Array<String> of CLI args"
            // but implementing this requires: (a) decoding the method's blob-heap signature to detect
            // param count, (b) allocating an Array<String> on the GC heap before the task starts.
            // For Phase 21, all entry methods are called with zero args via spawn_task(method_idx, vec![]).
        },
    }
    ```

    `fn main()` parses CLI, dispatches to handler functions, prints errors to stderr and exits with code 1 on error.

    **cmd_assemble(input, output):**
    - If input is "-", read stdin to string. Otherwise `std::fs::read_to_string(&input)`.
    - Call `writ_assembler::assemble(&src)`. On error, format error list and return Err.
    - Compute output path: if `--output` given, use it. Otherwise replace `.writil` extension with `.writc` (or append `.writc`).
    - Write `module.to_bytes()` to output file.
    - Print `Assembled: {output_path}` to stderr (informational).

    **cmd_disasm(input, verbose):**
    - Read file bytes: `std::fs::read(&input)`.
    - Parse: `writ_module::Module::from_bytes(&bytes)`. On error, return Err with message.
    - Call `writ_assembler::disassemble(&module)` or `writ_assembler::disassemble_verbose(&module)` based on `verbose` flag.
    - Print text to stdout.

    **cmd_run(input, entry, interactive, verbose):**
    - Read file bytes and parse Module (same as disasm).
    - Find entry export: iterate `module.export_defs`, resolve each name via `read_string`, find one matching `entry`. If not found, collect all export names and return error: `no exported method '{}' found. Available exports: [...]`.
    - Extract method index: `let method_idx = (export.item.0 & 0x00FF_FFFF) as usize - 1;` (convert 1-based token to 0-based index). Also verify `export.item_kind == 0` (method export).
    - Create CliHost: `CliHost::new(&module, interactive, verbose)`.
    - Build runtime: `RuntimeBuilder::new(module).with_host(cli_host).build()`. On error, return Err.
    - Spawn entry task: `runtime.spawn_task(method_idx, vec![])`. On error, return Err.
    - Tick loop:
      ```rust
      loop {
          match runtime.tick(0.0, ExecutionLimit::None) {
              TickResult::AllCompleted | TickResult::Empty => break,
              TickResult::TasksSuspended(pending) => {
                  // With CliHost all requests are handled synchronously in on_request,
                  // so this branch should not be reached. If it is, break with error.
                  eprintln!("warning: {} tasks suspended unexpectedly", pending.len());
                  break;
              }
              TickResult::ExecutionLimitReached => break,
          }
      }
      ```
    - If verbose, call `runtime.host().print_stats()` (need to add accessor — or if Runtime doesn't expose host(), handle stats differently. Check: Runtime has `pub(crate) host: H` — not public. If no public accessor, skip stats for now or add a field to track in the tick loop). Actually, looking at RESEARCH.md, `Runtime` does have host()/host_mut() accessors added in Phase 17. Check the actual code — the summary says "Runtime::host()/host_mut() accessors added for test inspection". Use `runtime.host().print_stats()` if they exist; if they are pub(crate), we need to either make them pub or skip stats. Pragmatic approach: if host() is not public, just count ticks in the main loop.

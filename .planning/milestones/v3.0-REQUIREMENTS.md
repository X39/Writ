# Requirements: Writ Compiler

**Defined:** 2026-03-02
**Core Value:** Correct, spec-compliant implementation at every layer — lowering matches Section 28 exactly, runtime matches the IL spec exactly — structured so each layer can be extended independently.

## v3.0 Requirements

Requirements for v3.0 — full compilation pipeline (name resolution, type checking, IL codegen). Each maps to roadmap phases.

### Name Resolution

- [x] **RES-01**: Compiler collects all top-level declarations (fn, struct, entity, enum, contract, impl, component, extern, const, global) across all files into a symbol table keyed by namespace-qualified name
- [x] **RES-02**: Compiler resolves `using` declarations (plain and alias forms) with correct scoping and conflict detection
- [x] **RES-03**: Compiler resolves qualified paths (`ns::Name`, `Enum::Variant`, `::root::Name`)
- [x] **RES-04**: Compiler enforces visibility rules (private, pub, type-private) including same-namespace cross-file visibility
- [x] **RES-05**: Compiler resolves every `AstType` to a TypeRef blob or primitive tag, including writ-runtime virtual module types
- [x] **RES-06**: Compiler associates impl blocks with their target type and contract definitions
- [x] **RES-07**: Compiler scopes generic type parameters to their declaration and body, with correct shadowing
- [x] **RES-08**: Compiler resolves `self`/`mut self` in method bodies to the enclosing type
- [x] **RES-09**: Compiler validates `@Speaker` names resolve to `[Singleton]` entities with Speaker component
- [x] **RES-10**: Compiler validates `[Singleton]` and `[Conditional]` attributes target allowed declaration kinds
- [x] **RES-11**: Compiler detects ambiguous names from multiple `using` imports with both candidate spans
- [x] **RES-12**: Compiler suggests similar names on resolution failure ("did you mean `survival::HealthPotion`?")

### Type Checking

- [x] **TYPE-01**: Compiler infers primitive types for literals (int, float, bool, string)
- [x] **TYPE-02**: Compiler infers `let` binding types from initializer expressions; checks annotated bindings for compatibility
- [x] **TYPE-03**: Compiler checks function call arity, argument types, and propagates return types
- [x] **TYPE-04**: Compiler checks field access types, distinguishing struct/entity script fields from component fields
- [x] **TYPE-05**: Compiler verifies concrete type arguments satisfy contract bounds at generic call sites
- [x] **TYPE-06**: Compiler enforces strict mutability (`let` prevents reassignment AND mutation; `mut self` vs `self`)
- [x] **TYPE-07**: Compiler verifies all code paths return the declared type; void functions don't return values
- [x] **TYPE-08**: Compiler checks `?` requires Option scrutinee and Option return context; `!` works on Option and Result
- [x] **TYPE-09**: Compiler checks `try` requires Result scrutinee and compatible Result return context
- [x] **TYPE-10**: Compiler checks enum `match` exhaustiveness (all variants covered or wildcard present)
- [x] **TYPE-11**: Compiler types component access as guaranteed (concrete entity) or Optional (generic Entity)
- [x] **TYPE-12**: Compiler infers closure captures, classifies by-value vs by-reference, and checks immutability constraints
- [x] **TYPE-13**: Compiler infers generic type arguments at call sites via unification
- [x] **TYPE-14**: Compiler checks `spawn` returns TaskHandle, `join` returns result type, `cancel` is void
- [x] **TYPE-15**: Compiler checks `new Type {}` for field presence, field types, and entity vs struct distinction
- [x] **TYPE-16**: Compiler binds `for` loop variable type via `Iterable<T>` impl lookup
- [x] **TYPE-17**: Compiler desugars `?` and `!` operators to typed match nodes in the typed IR
- [x] **TYPE-18**: Compiler produces precise mutability errors pointing to both mutation site and immutable binding
- [x] **TYPE-19**: Compiler suggests missing contract implementations in type errors

### IL Code Generation

- [x] **EMIT-01**: Compiler emits ModuleDef, ModuleRef (including writ-runtime), and ExportDef metadata
- [x] **EMIT-02**: Compiler emits TypeDef, FieldDef, MethodDef, and ParamDef rows for all named types
- [x] **EMIT-03**: Compiler emits ContractDef, ContractMethod, and ImplDef rows with correct CALL_VIRT slot ordering from contract declaration
- [x] **EMIT-04**: Compiler emits GenericParam and GenericConstraint rows
- [x] **EMIT-05**: Compiler emits GlobalDef and ExternDef rows
- [x] **EMIT-06**: Compiler emits ComponentSlot rows with override blobs for entity component slots
- [x] **EMIT-07**: Compiler assigns register indices via linear allocation and emits per-register TypeRef table
- [x] **EMIT-08**: Compiler emits all arithmetic, logic, comparison, data movement, and control flow instructions
- [x] **EMIT-09**: Compiler emits CALL, CALL_VIRT, CALL_EXTERN, and CALL_INDIRECT with correct dispatch selection
- [x] **EMIT-10**: Compiler emits NEW, GET_FIELD, SET_FIELD for struct construction and field access
- [x] **EMIT-11**: Compiler emits SPAWN_ENTITY, INIT_ENTITY, DESTROY_ENTITY, ENTITY_IS_ALIVE, GET_COMPONENT, GET_OR_CREATE, FIND_ALL per spec entity construction sequence
- [x] **EMIT-12**: Compiler emits all 9 array instructions
- [x] **EMIT-13**: Compiler emits all 10 Option/Result instructions
- [x] **EMIT-14**: Compiler emits closure/delegate with compiler-generated capture struct TypeDef, method body, and NEW_DELEGATE
- [x] **EMIT-15**: Compiler emits SPAWN_TASK, SPAWN_DETACHED, JOIN, CANCEL, DEFER_PUSH/POP/END
- [x] **EMIT-16**: Compiler emits ATOMIC_BEGIN/ATOMIC_END for atomic blocks
- [x] **EMIT-17**: Compiler emits GET_TAG + SWITCH + EXTRACT_FIELD for enum pattern matching
- [x] **EMIT-18**: Compiler emits NEW_ENUM with correct variant tag and payload registers
- [x] **EMIT-19**: Compiler emits I2F, F2I, I2S, F2S, B2S, CONVERT for type conversions
- [x] **EMIT-20**: Compiler emits STR_CONCAT, STR_BUILD, STR_LEN for string operations
- [x] **EMIT-21**: Compiler emits BOX/UNBOX at generic call sites where value types are passed
- [x] **EMIT-22**: Compiler registers lifecycle hook MethodDefs with correct hook_kind flags in entity TypeDefs
- [x] **EMIT-23**: Compiler emits IS_NONE/IS_ERR + early return sequences for `?` and `try` propagation
- [x] **EMIT-24**: Compiler emits TAIL_CALL for dialogue transition returns
- [x] **EMIT-25**: Compiler emits LocaleDef table rows for localization keys
- [x] **EMIT-26**: Compiler emits SourceSpan and DebugLocal entries for debug info
- [x] **EMIT-27**: Compiler specializes CALL_VIRT to CALL when receiver's static type is known concrete
- [x] **EMIT-28**: Compiler folds constant arithmetic expressions in `const` declarations
- [x] **EMIT-29**: Compiler emits AttributeDef rows for runtime attribute inspection

### CLI Integration

- [x] **CLI-01**: `writ compile` subcommand accepts .writ source file(s) and outputs .writil binary module
- [x] **CLI-02**: Compiler pipeline runs end-to-end: parse → lower → resolve → typecheck → codegen → write module
- [x] **CLI-03**: Compilation errors display with source spans, multi-span context, and actionable messages

### Runtime Gap Closure

- [x] **FIX-01**: Runtime dispatches lifecycle hooks (on_create, on_destroy, on_interact, on_finalize) via method name lookup
- [x] **FIX-02**: Runtime resolves generic contract specialization without collision (e.g. Int:Into<Float> vs Int:Into<String>)
- [x] **FIX-03**: CliHost dereferences GC heap Ref values for display (strings, objects)

## Out of Scope

| Feature | Reason |
|---------|--------|
| Implicit type coercions | Spec §10.2 explicitly forbids — callers must use `.into<T>()` |
| Full HM type inference for declarations | Spec §5.2 requires explicit annotations on all declarations |
| Structural subtyping / duck typing | Spec uses nominal typing — explicit `impl` blocks required |
| Register reuse / liveness analysis | Premature optimization for reference interpreter; defer to JIT |
| Optimization passes (DCE, CSE, inlining) | Correctness-first milestone; optimization is a JIT concern |
| Stack machine output | IL spec is register-based (§2.1) |
| Re-export via `using` | Spec §23.4 explicitly forbids re-export |
| Glob imports (`using ns::*`) | Not in spec; creates invisible dependencies |
| Cross-function escape analysis for closures | GC handles closure lifetime; escape analysis is a JIT concern |

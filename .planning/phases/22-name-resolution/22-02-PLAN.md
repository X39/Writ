<objective>
Implement Pass 2 of name resolution: the body resolver and scope chain. This pass walks every declaration body in the AST, resolving all identifier references (types, values, paths) against the DefMap populated by Plan 01's collector. It produces a fully resolved `NameResolvedAst` where every type reference is a `ResolvedType` and every name reference carries a `DefId`.

Purpose: Pass 2 is the core of name resolution. After this plan, every identifier in every source file is bound to either a DefId (user declaration), a primitive tag, or a prelude type. Errors are emitted for unresolved names, visibility violations, and ambiguous imports. This directly satisfies 7 of the 12 phase requirements.

Output: Complete resolver that handles using imports, qualified paths, visibility enforcement, type resolution, impl association, generic scoping, and self resolution. Snapshot tests verify resolution output for each capability.
</objective>

    ```rust
    pub struct ScopeChain<'def> {
        def_map: &'def DefMap,
        current_file: FileId,
        current_ns: String,
        /// Active using imports: (alias_or_terminal, target_fqn, using_span)
        active_usings: Vec<UsingEntry>,
        /// Stack of scope layers (generic params, fn params, local blocks)
        layers: Vec<ScopeLayer>,
        /// Current self type (inside impl/entity method)
        self_type: Option<DefId>,
    }
    ```

    **UsingEntry struct:**
    - `alias: String` — the local name (either explicit alias or terminal path segment)
    - `target_ns: String` — the namespace being imported (for `using survival;` → target_ns = "survival")
    - `target_fqn: Option<String>` — for `using survival::HealthPotion;` → target_fqn = "survival::HealthPotion"
    - `span: SimpleSpan` — for error reporting
    - `used: bool` — initially false; set to true when this using entry resolves a name. After resolve_bodies completes for a file, any UsingEntry with `used == false` emits W0001 (unused import) warning diagnostic.

    **ScopeLayer enum:**
    - `GenericParams(Vec<(String, SimpleSpan)>)` — generic type param names
    - `Locals(Vec<(String, SimpleSpan)>)` — local let bindings (value names, not types — type resolution doesn't use this)

    **resolve_type method:**
    Lookup order for unqualified type names:
    1. Check generic params (innermost layer first, walk stack backwards)
    2. Check primitives via `is_primitive_name()`
    3. Check prelude types/contracts via prelude lookup
    4. Check current namespace (same-file private + cross-file pub) in `def_map.by_fqn` using FQN `current_ns::name` AND check file_private
    5. Check using imports — for each active using:
       - If using targets a specific FQN (e.g., `using survival::HealthPotion;`): check if alias matches
       - If using targets a namespace (e.g., `using survival;`): check `def_map.by_fqn` for `target_ns::name`
       - Collect all matches. If count > 1 → ambiguity error (RES-11). If count == 1 → check visibility.
       - **When a using entry successfully resolves a name**, set `entry.used = true` on the matched UsingEntry (requires `&mut self` on resolve_type/resolve_value or interior mutability via Cell<bool>).
    6. Check root namespace (`def_map.by_fqn` with just `name` — for root-level declarations)
    7. Not found → return error (unresolved, handled by Plan 03 for fuzzy suggestions)

    **resolve_qualified_path method:**
    For `a::b::c`:
    1. Resolve first segment `a`: is it a namespace? Check `def_map.namespace_members`. Is it a type (enum)? Check `def_map.by_fqn`.
    2. If namespace: resolve `a::b` recursively. For final segment, look up in that namespace.
    3. If enum type: remaining segments are variant names. Return the variant path (DefId of enum + variant name). Variant-level resolution stores the enum DefId; the type checker validates variants exist.
    4. Root-anchored (`::a::b`): skip current namespace and start from root.

    **resolve_visibility check:**
    When a DefId is found, before returning success:
    - If `def.vis == Private` AND `def.file_id != current_file` → emit E0005 VisibilityViolation.
    - For cross-namespace access: pub required always.
    - For same-namespace cross-file: pub required (private is file-local per spec).

    **resolve_using_decl method:**
    For each `AstUsingDecl`:
    - `path` is `["survival"]` (namespace import) or `["survival", "HealthPotion"]` (specific import).
    - If path has one segment: it's a namespace import. Validate namespace exists in def_map.namespace_members.
    - If path has multiple segments: check if the full path resolves to a definition (specific import) or if all-but-last is a namespace (namespace import with terminal).
    - Create `UsingEntry` and push to `active_usings`.
    - Detect scope: if we're inside a block namespace, the using is block-scoped. If file-level, file-scoped. (Track via a using_scope_depth counter that increments on namespace block entry.)

    **emit_unused_import_warnings method:**
    After all declarations in a file have been resolved, call `scope.emit_unused_import_warnings(diags)` to iterate `active_usings` and emit W0001 for any entry where `used == false`. The diagnostic should include the using's span and help: "remove this unused import".

    - `UnusedImport { alias, span }` — W0001 warning: "unused import `{alias}`", help: "remove this unused import"
    - Add `impl From<ResolutionError> for Diagnostic` for each new variant with specific error codes, labels, and help text.

    **resolve_bodies function:**
    ```rust
    pub fn resolve_bodies(
        asts: &[(FileId, Ast)],
        def_map: &DefMap,
    ) -> (Vec<ResolvedDecl>, Vec<Diagnostic>)
    ```

    For each file's AST:
    1. Create a ScopeChain for the file with current_file, current_ns.
    2. Process file-level `using` declarations first (push to scope's active_usings with `used: false`).
    3. Walk each AstDecl:
       - **AstDecl::Fn**: push generic params → resolve param types → set return type → resolve body statements → pop generics. Produce `ResolvedDecl::Fn` with resolved types.
       - **AstDecl::Struct**: push generics → resolve field types → resolve hook bodies → pop. Produce `ResolvedDecl::Struct`.
       - **AstDecl::Entity**: push generics (none for entities, but handle uniformly) → resolve property types → resolve component slot names (check DefKind == Component or ExternComponent; emit error if not) → set self_type → resolve hook bodies → resolve inherent impl methods → pop. Produce `ResolvedDecl::Entity`.
       - **AstDecl::Enum**: push generics → resolve variant field types → pop. Produce `ResolvedDecl::Enum`.
       - **AstDecl::Contract**: push generics → resolve member signatures → pop. Produce `ResolvedDecl::Contract`.
       - **AstDecl::Impl**: resolve target type → resolve contract type if present → associate impl with target DefId (store mapping in a side table `impl_targets: Vec<(DefId, DefId, Option<DefId>)>` — impl_id, target_id, contract_id) → push generics → set self_type → resolve method bodies → pop generics → clear self_type. Produce `ResolvedDecl::Impl`.
       - **AstDecl::Const / AstDecl::Global**: resolve type annotation → resolve value expression.
       - **AstDecl::Extern(Fn/Struct/Component)**: resolve type references in signatures/fields.
       - **AstDecl::Namespace(Block { items, .. })**: push using scope depth, push namespace, recurse items, pop.
    4. **After all declarations in a file are resolved**, call `scope.emit_unused_import_warnings(&mut diags)` to emit W0001 for any UsingEntry with `used == false`.

    **resolve_type helper:**
    Maps `AstType` to `ResolvedType`:
    - `Named { name, .. }` → call `scope.resolve_type(name)` → on success, `ResolvedType::Named { def_id, type_args: vec![] }`; on failure, emit diagnostic, return `ResolvedType::Error`.
    - `Generic { name, args, .. }` → resolve outer name, then resolve each arg recursively → `ResolvedType::Named { def_id, type_args }`.
    - `Array { elem, .. }` → resolve elem → `ResolvedType::Array(Box::new(resolved_elem))`.
    - `Func { params, ret, .. }` → resolve each param, resolve ret → `ResolvedType::Func { ... }`.
    - `Void { .. }` → `ResolvedType::Void`.

    **resolve_expr helper:**
    Maps `AstExpr` (in-place annotation or side-table). For Phase 22, expression resolution focuses on:
    - `Ident { name, .. }` → look up in scope layers (locals, params) then in def_map as a value (fn, const, global). Store resolution result.
    - **`Null { .. }` (or `Ident { name: "null", .. }`)** → resolve to `Option::None`. `null` is a hard keyword per CONTEXT.md that always resolves to `Option::None`. In the resolved IR, represent as a special `ResolvedExpr::Null` variant or as an enum variant access `Option::None`. The resolver should NOT look up "null" in the DefMap; it is handled as a built-in keyword that maps to the prelude Option type's None variant.
    - `Path { segments, .. }` → call scope.resolve_qualified_path.
    - `SelfLit { .. }` → check scope.self_type is Some; if None, emit error.
    - `New { ty, .. }` → resolve ty.
    - `MemberAccess`, `Call`, `GenericCall`, `BracketAccess` → resolve sub-expressions recursively. (Field/method resolution is deferred to type checking; name resolution just resolves the base expression.)
    - All other expression variants: recurse into sub-expressions and sub-statements.

    **resolve_stmt helper:**
    - `Let { ty, value, .. }` → resolve type annotation if present, resolve value expression, push binding name to scope's Locals layer.
    - `For { iterable, body, .. }` → resolve iterable, push binding, resolve body, pop binding.
    - `While / If / Match / Block` → recurse.
    - `Return { value, .. }` → resolve value if present.

    **resolve_pattern helper:**
    - `EnumDestructure { path, fields, .. }` → resolve path to enum DefId via scope.resolve_qualified_path. Recurse into sub-patterns.
    - Other pattern variants: pass through.

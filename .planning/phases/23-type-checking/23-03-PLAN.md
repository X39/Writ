From CONTEXT.md - ? desugaring:
- `expr?` where expr: Option<T>, enclosing fn returns Option<U>
  -> Match { scrutinee: expr, arms: [Some(x) -> x, None -> Return(None)] }
- `expr!` where expr: Option<T> or Result<T,E>
  -> Match { scrutinee: expr, arms: [Some/Ok(x) -> x, None/Err -> crash/panic] }
- `try expr` where expr: Result<T,E>, enclosing fn returns Result<U,E>
  -> Match { scrutinee: expr, arms: [Ok(x) -> x, Err(e) -> Return(Err(e))] }

    **Create writ-compiler/src/check/desugar.rs:**

    `desugar_question(ctx: &mut CheckCtx, expr: &AstExpr, inner_expr: &AstExpr, span: SimpleSpan) -> TypedExpr`:
    - Check inner_expr, get its type
    - If type is `TyKind::Option(inner_ty)`:
      - Verify `ctx.current_fn_ret` is `TyKind::Option(_)`. If not, emit E0114 "? operator requires function to return Option<_>"
      - Produce `TypedExpr::Match { ty: inner_ty, scrutinee: typed_inner, arms: [Some_arm, None_arm] }`
      - Some arm: `TypedArm { pattern: EnumVariant { variant_name: "Some", bindings: [Variable { name: "__val", ty: inner_ty }] }, body: TypedExpr::Var { ty: inner_ty, name: "__val" } }`
      - None arm: `TypedArm { pattern: EnumVariant { variant_name: "None", bindings: [] }, body: TypedExpr::Return { ty: ctx.interner.void(), value: Some(Box::new(TypedExpr::Path { ty: current_fn_ret, segments: vec!["None"] })) } }`
    - If not Option: emit E0113 "? operator requires Option<T>, found X"
    - Return the Match or Error

    `desugar_unwrap(ctx: &mut CheckCtx, expr: &AstExpr, inner_expr: &AstExpr, span: SimpleSpan) -> TypedExpr`:
    - Check inner_expr, get its type
    - If `TyKind::Option(inner_ty)`:
      - Produce Match: Some -> unwrap, None -> crash (Error or a special Crash node)
      - Match ty is `inner_ty`
    - If `TyKind::Result(ok_ty, _)`:
      - Produce Match: Ok -> unwrap, Err -> crash
      - Match ty is `ok_ty`
    - Otherwise: emit error "! operator requires Option or Result"

    `desugar_try(ctx: &mut CheckCtx, expr: &AstExpr, inner_expr: &AstExpr, span: SimpleSpan) -> TypedExpr`:
    - Check inner_expr, get its type
    - If `TyKind::Result(ok_ty, err_ty)`:
      - Verify `ctx.current_fn_ret` is `TyKind::Result(_, err_ty2)` where err_ty unifies with err_ty2. If not, emit E0115
      - Produce Match: Ok(val) -> val, Err(e) -> Return(Err(e))
      - Match ty is `ok_ty`
    - Otherwise: emit error "try requires Result<T, E>"

    **Implement lambda/closure checking in check_expr.rs (replace Lambda stub):**
    - `Lambda { params, return_type, body, span }`:
      - Determine param types: if each `AstLambdaParam` has an explicit type annotation, use it. If not, check for expected type context (bidirectional inference):
        - If the lambda is being passed as a function argument, and the parameter type is `Func { params, ret }`, use those types for unannotated lambda params
        - If no context available and no annotation, create InferVar for each param
      - Determine return type: explicit annotation or infer from expected context or InferVar
      - Push scope, define params in local_env
      - **Capture inference**: before checking the body, snapshot the current local_env. After checking the body, identify all variables referenced in the body that were NOT defined in the lambda's own scope (they come from outer scopes). For each such variable:
        - Look up its mutability in the outer local_env
        - `Mutability::Immutable` -> `CaptureMode::ByValue` (spec section 12.4.4)
        - `Mutability::Mutable` -> `CaptureMode::ByRef`
        - Store as `Capture { name, ty, mode, binding_span }`
      - Pop scope
      - Build the Func type: `TyKind::Func { params: param_tys, ret: ret_ty }`
      - Return `TypedExpr::Lambda { ty: func_ty, params, ret_ty, captures, body }`

    **Capture inference implementation detail:**
    To track which variables are captured (referenced but not locally defined), add a mechanism to LocalEnv:
    - Add a `capture_boundary: Option<usize>` field that records the scope depth at which a lambda starts
    - When `capture_boundary` is set and a variable lookup resolves to a scope below the boundary, record it as a capture
    - Alternative approach: after type-checking the lambda body, diff the variable accesses against the lambda's local definitions. Variables accessed but not in the lambda's scope are captures
    - Choose whichever approach is cleaner. The result is a `Vec<Capture>` stored in the TypedExpr::Lambda

    **Immutability checking inside closures:**
    - If a capture is ByValue (from `let` binding), attempting to mutate it inside the closure produces E0107
    - If a capture is ByRef (from `let mut` binding), mutation is allowed
    - This reuses the existing mutability checking from Plan 23-02's mutability.rs

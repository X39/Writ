    - ContractDef.method_list points to first ContractMethod row for that contract

  <action>
  1. Create `writ-compiler/src/emit/slots.rs`:
     - `pub fn assign_vtable_slots(builder: &mut ModuleBuilder)`
     - Walk all ContractDef entries in the builder. For each contract's ContractMethod rows (found via method_list range), assign slot indices 0, 1, 2... in the ORDER the ContractMethods were added (which is contract declaration order from the AST).
     - This is the critical invariant: slots come from contract declaration order.

  2. Extend `collect.rs` to handle `TypedDecl::Contract { def_id }`:
     - Look up DefEntry from DefMap. Find matching AST contract decl (reuse find_contract_decl pattern from env.rs).
     - Call `builder.add_contract_def(name, namespace)` to get a contract handle.
     - For each contract member (AstContractMember::FnSig):
       - Encode the method signature as a blob (param types + return type).
       - Call `builder.add_contract_method(contract_handle, name, sig_blob, slot=0)` — slot assigned later by slots.rs.
     - For contract generics: emit GenericParam rows with owner = ContractDef token.
     - For generic constraints on contract type params: emit GenericConstraint rows.
     - Track DefId -> contract handle mapping.

  3. Extend `collect.rs` to handle `TypedDecl::Impl { def_id, methods }`:
     - Look up DefEntry from DefMap. Find matching AST impl decl.
     - Resolve target type DefId from impl_decl.target. Resolve contract DefId from impl_decl.contract (if any).
     - Call `builder.add_impl_def(type_token, contract_token)` to get an impl handle.
     - For each method (method_def_id, _body) in TypedDecl::Impl.methods:
       - The MethodDef was already emitted in Plan 24-01 under the target type's TypeDef.
       - Link the impl's method_list to these MethodDef rows. The impl needs to know which MethodDefs correspond to its methods.
       - NOTE: If Plan 24-01 emits impl methods under the target TypeDef, then ImplDef.method_list should point to those same MethodDef rows. If impl method MethodDefs need to be separate from inherent method MethodDefs, adjust accordingly.
     - For contract impls: each impl method must be mapped to its corresponding ContractMethod slot by name matching.

  4. Update `writ-compiler/src/emit/mod.rs`:
     - After `collect::collect_defs()`, call `slots::assign_vtable_slots(&mut builder)`.
     - Then call `builder.finalize()`.

  5. Flesh out ModuleBuilder methods that were stubs in Plan 24-01:
     - `add_contract_def(&mut self, name: &str, namespace: &str) -> ContractHandle`
     - `add_contract_method(&mut self, contract: ContractHandle, name: &str, sig_blob: u32, slot: u16) -> usize`
     - `add_impl_def(&mut self, type_token: MetadataToken, contract_token: MetadataToken) -> ImplHandle`
     - Update `finalize()` to:
       - Group ContractMethod rows by parent ContractDef, assign contiguous row indices, set ContractDef.method_list.
       - Assign ImplDef row indices, set ImplDef.method_list pointing to the correct MethodDef range.

  6. Add tests in `emit_tests.rs`:
     - Test contract emission with 2 methods -> verify ContractDef + 2 ContractMethod rows.
     - Test slot ordering: contract declares methods A, B, C. Impl declares them in order C, A, B. Verify ContractMethod slots are A=0, B=1, C=2 regardless of impl order.
     - Test ImplDef links: verify type_token and contract_token are correct MetadataTokens.
     - Test generic contract: verify GenericParam rows attached to ContractDef.

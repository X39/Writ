1. **Create `writ-compiler/src/emit/body/call.rs`**:

   - `pub fn emit_call(emitter: &mut BodyEmitter, call_expr: &TypedExpr) -> u16`: Main entry dispatching on callee analysis.

   - **Callee analysis** — Determine call kind by examining the callee expression:
     ```
     TypedExpr::Call { callee, args, ty, .. }
       callee = TypedExpr::Var { name, .. } with DefId resolving to:
         DefKind::Fn -> CallKind::Direct(method_token)
         DefKind::ExternFn -> CallKind::Extern(extern_token)
       callee = TypedExpr::Field { receiver, field, .. }:
         Check if field resolves to a contract method (via DefMap/ImplDef lookup):
           Yes + receiver TyKind::Struct/Entity (concrete) -> CallKind::Direct (EMIT-27)
           Yes + receiver is generic -> CallKind::Virtual(slot)
         Otherwise -> CallKind::Direct(method_token)
       callee = TypedExpr with TyKind::Func -> CallKind::Indirect
     ```

   - **Argument packing** (CRITICAL — Pitfall 7 from RESEARCH.md):
     1. Emit each argument expression, collecting result registers into `Vec<u16>`
     2. Allocate a consecutive block of `argc` registers starting at `regs.next`
     3. MOV each emitted arg register into its consecutive slot: `MOV r_slot, r_emitted` (skip if already consecutive)
     4. For method calls, the receiver object goes in r_base (it's the implicit `self`)

   - **Boxing logic** (EMIT-21):
     - For each argument: if arg's TyKind is `Int|Float|Bool|Enum` AND the corresponding parameter type is `GenericParam`, emit `Box { r_dst: new_reg, r_val: arg_reg }` and use the boxed register in the consecutive block
     - After the call: if the call's return TyKind at the *call site* is a value type but the method's declared return is GenericParam, emit `Unbox { r_dst: new_reg, r_boxed: call_result_reg }`

   - **Emit the call instruction:**
     ```rust
     match call_kind {
         CallKind::Direct(token) => Instruction::Call { r_dst, method_idx: token.0, r_base, argc },
         CallKind::Virtual(slot) => Instruction::CallVirt { r_dst, slot, r_obj: r_base, r_base: r_base + 1, argc: argc - 1 },
         CallKind::Extern(token) => Instruction::CallExtern { r_dst, extern_idx: token.0, r_base, argc },
         CallKind::Indirect => Instruction::CallIndirect { r_dst, r_delegate: callee_reg, r_base, argc },
     }
     ```

   - **Note on callee DefId resolution:** The callee's DefId must be extracted from the TypedExpr. For `TypedExpr::Var`, look up the name in DefMap or use the DefId embedded in resolution context. For `TypedExpr::Field`, the method's DefId comes from the receiver type's impl block. Use `builder.token_for_def()` to get MetadataTokens. If the name cannot be found in `def_token_map` (e.g., it's a writ-runtime builtin), emit a placeholder or use a MethodRef token.

1. **Update `writ-compiler/src/emit/body/expr.rs`** — Replace TypedExpr::New placeholder:

   ```rust
   TypedExpr::New { ty, target_def_id, fields, span } => {
       let type_token = emitter.builder.token_for_def(*target_def_id)
           .expect("New target should have a MetadataToken");
       match emitter.interner.kind(*ty) {
           TyKind::Entity(_) => {
               // EMIT-11: Entity construction sequence per spec §2.16.7
               let r_entity = emitter.regs.alloc(*ty);
               emitter.instructions.push(Instruction::SpawnEntity {
                   r_dst: r_entity,
                   type_idx: type_token.0,
               });
               // ONLY explicitly-provided fields get SET_FIELD
               for (field_name, field_expr) in fields {
                   let r_val = emit_expr(emitter, field_expr);
                   let field_token = resolve_field_token(emitter, *target_def_id, field_name);
                   emitter.instructions.push(Instruction::SetField {
                       r_obj: r_entity,
                       field_idx: field_token,
                       r_val,
                   });
               }
               emitter.instructions.push(Instruction::InitEntity { r_entity });
               r_entity
           }
           _ => {
               // EMIT-10: Struct construction
               let r_obj = emitter.regs.alloc(*ty);
               emitter.instructions.push(Instruction::New {
                   r_dst: r_obj,
                   type_idx: type_token.0,
               });
               for (field_name, field_expr) in fields {
                   let r_val = emit_expr(emitter, field_expr);
                   let field_token = resolve_field_token(emitter, *target_def_id, field_name);
                   emitter.instructions.push(Instruction::SetField {
                       r_obj: r_obj,
                       field_idx: field_token,
                       r_val,
                   });
               }
               r_obj
           }
       }
   }
   ```

   - **resolve_field_token helper:** Look up the field's DefId by searching the TypeDef's field list in the builder, then get its MetadataToken. Since fields don't have DefIds directly, use the field name + parent TypeDef handle to find the field index. The field_idx in SET_FIELD/GET_FIELD is the MetadataToken.0 (encoded token) for the FieldDef row, computed as `MetadataToken::new(TableId::FieldDef, row_index).0`. Helper function in `body/mod.rs` that iterates builder field_defs to find the field by parent + name.

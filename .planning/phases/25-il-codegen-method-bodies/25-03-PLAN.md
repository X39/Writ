1. **Create `writ-compiler/src/emit/body/closure.rs`** (EMIT-14):

   - **`pre_scan_lambdas(typed_ast, interner, builder) -> Vec<LambdaInfo>`:**
     - Walk all `TypedDecl::Fn.body`, `TypedDecl::Impl.methods[*].body`, `TypedDecl::Const.value`, `TypedDecl::Global.value` recursively
     - For each `TypedExpr::Lambda` found, assign a unique closure index (e.g., `__closure_0`, `__closure_1`)
     - Register a synthetic `TypeDef` for the capture struct:
       ```rust
       let type_handle = builder.add_typedef(
           &format!("__closure_{}", idx),
           "",  // no namespace
           TypeDefKind::Struct,
           0,   // flags
           None, // no DefId
       );
       for capture in &lambda.captures {
           let type_sig = type_sig::encode_type(capture.ty, interner, ...);
           let sig_offset = builder.blob_heap.intern(&type_sig);
           builder.add_fielddef(type_handle, &capture.name, sig_offset, 0);
       }
       ```
     - Register a synthetic `MethodDef` for the closure body:
       ```rust
       let method_handle = builder.add_methoddef(
           Some(type_handle),
           &format!("__invoke_{}", idx),
           signature_blob_offset,
           METHOD_FLAG_STATIC, // or 0
           None, // no DefId — use a synthetic marker
       );
       ```
     - Return `LambdaInfo { closure_idx, type_handle, method_handle, captures, params, body, ret_ty }`

   - **Timing (CRITICAL — Pitfall 2 from RESEARCH.md):**
     - `pre_scan_lambdas()` MUST be called BEFORE `builder.finalize()`.
     - Restructure the pipeline in `emit/mod.rs`:
       ```rust
       pub fn emit(...) -> (ModuleBuilder, Vec<Diagnostic>) {
           // Phase 24 work (unchanged)
           collect::collect_defs(...);
           slots::assign_vtable_slots(&mut builder);
           // Phase 25 addition: register closure TypeDefs BEFORE finalize
           let lambda_infos = closure::pre_scan_lambdas(typed_ast, interner, &mut builder);
           builder.finalize();
           collect::collect_post_finalize(...);
           (builder, diags)
       }
       ```
     - The `lambda_infos` are passed to `emit_all_bodies()` for use during body emission.

   - **`emit_lambda(emitter, lambda_expr, lambda_info) -> u16`:**
     - If captures are empty:
       ```
       LOAD_NULL  r_null
       NEW_DELEGATE  r_delegate, closure_method_token, r_null
       ```
     - If captures present:
       ```
       NEW          r_env, capture_struct_type_token
       // For each capture:
       MOV/GET_FIELD  r_captured, <resolve capture from locals>
       SET_FIELD    r_env, capture_field_token, r_captured
       NEW_DELEGATE r_delegate, closure_method_token, r_env
       ```
     - The closure body is emitted as a separate `EmittedBody` entry (create a new `BodyEmitter`, set up capture fields as params, emit the lambda body, collect instructions).

2. **Update `writ-compiler/src/emit/body/expr.rs`** — Replace Lambda placeholder:
   - Look up the `LambdaInfo` for this specific lambda (by matching on the TypedExpr identity or a lambda counter)
   - Call `closure::emit_lambda(emitter, expr, &lambda_info)`

3. **Concurrency instructions** (EMIT-15) — update expr.rs:
   - **TypedExpr::Spawn:** The spawn expr should be a Call expression. Emit the call arguments, then:
     ```
     SPAWN_TASK { r_dst, method_idx, r_base, argc }
     ```
     Extract method_idx from the inner call's callee.
   - **TypedExpr::SpawnDetached:** Same as Spawn but emit `SpawnDetached` instruction.
   - **TypedExpr::Join:** `emit expr -> r_task`, emit `Join { r_dst, r_task }`
   - **TypedExpr::Cancel:** `emit expr -> r_task`, emit `Cancel { r_task }`
   - **TypedExpr::Defer:**
     - Create a defer_handler label
     - Emit `DeferPush { r_dst: 0, method_idx: 0 }` (placeholder offset, fixed up)
     - Add fixup for the DeferPush handler offset
     - Emit `DeferPop` immediately after the scope block
     - At the end of the method body (or a deferred section), mark the defer_handler label, emit the defer body, emit `DeferEnd`
     - **Note on DEFER_PUSH handler_offset:** Per Pitfall 6 from RESEARCH.md, the `method_idx` field in `DeferPush` actually stores the byte offset to the handler code. Use label fixup to resolve this.

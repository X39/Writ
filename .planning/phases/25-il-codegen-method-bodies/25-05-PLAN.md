**SWITCH fixup fix (patterns.rs):**

The current approach uses a label-based fixup system designed for single-offset branch instructions (BR/BR_FALSE etc.), which writes the offset at bytes [4..8) of the branch instruction. SWITCH is different: it has a variable-length `offsets: Vec<i32>` where each slot needs its own offset value.

The fix: Instead of using add_fixup() (which patches byte offsets), directly update the SWITCH instruction's `offsets` Vec after all variant arms are emitted and labels are marked. The labels system's `mark()` stores instruction-index positions. After all arms are emitted, iterate arm_labels, get each label's resolved instruction-index position, and compute the relative offset from the SWITCH instruction position.

Replace the no-op loop at lines 99-106 of patterns.rs with:

```rust
// After all variant arms and wildcard arm are emitted, patch SWITCH offsets.
// Each arm_labels[i] has been marked via mark_label_here with an instruction index.
// Compute offset = arm_label_pos - switch_idx for each slot.
// Also handle fallthrough to wildcard by marking its label if tag >= n_variants.

// NOTE: Cannot use add_fixup for SWITCH because it has multiple offset slots.
// Instead, after all arms are emitted and labels are marked, directly patch
// the offsets vec in the Switch instruction at switch_idx.

// (Move this block to AFTER all arms and wildcard are emitted, just before mark end_label)
let mut patched_offsets: Vec<i32> = Vec::with_capacity(n_variants);
for label in &arm_labels {
    let target_pos = emitter.labels.resolved.get(&label.0).copied().unwrap_or(0);
    patched_offsets.push((target_pos as i64 - switch_idx as i64) as i32);
}
// Patch the Switch instruction
if let Instruction::Switch { offsets, .. } = &mut emitter.instructions[switch_idx] {
    *offsets = patched_offsets;
}
```

However, the `resolved` field on LabelAllocator is currently private. Either:
(a) Make `resolved` pub(crate), or
(b) Add a `resolve(&self, label: Label) -> Option<usize>` public method to LabelAllocator.

Option (b) is cleaner. Add to labels.rs:
```rust

    self.resolved.get(&label.0).copied()
}
```

Then in the SWITCH fixup code, use `emitter.labels.resolve(label)` instead of direct field access.

IMPORTANT: The SWITCH fixup code must be placed AFTER all variant arms and the wildcard arm have been emitted and their labels marked. Currently the no-op loop is at lines 99-106 which is BEFORE the arms are emitted. Move the fixup to just before `emitter.mark_label_here(end_label)` at line 151.

In `emit_all_bodies()`, the match on TypedDecl currently only handles `Fn` and `Impl`, with `_ => {}` catching everything else. Add a `TypedDecl::Const` arm:

```rust

    let mut emitter = BodyEmitter::new(builder, interner);
    emitter.current_method_def_id = Some(*def_id);
    // Try constant folding first
    if let Some(folded) = const_fold::const_fold(value, interner) {
        // Emit a single load instruction for the folded constant
        let r = match &folded {
            TypedLiteral::Int(v) => {
                let r = emitter.alloc_reg(Ty(0)); // Int
                emitter.emit(Instruction::LoadInt { r_dst: r, value: *v });
                r
            }
            TypedLiteral::Float(v) => {
                let r = emitter.alloc_reg(Ty(1)); // Float
                emitter.emit(Instruction::LoadFloat { r_dst: r, value: *v });
                r
            }
            TypedLiteral::Bool(true) => {
                let r = emitter.alloc_reg(Ty(2)); // Bool
                emitter.emit(Instruction::LoadTrue { r_dst: r });
                r
            }
            TypedLiteral::Bool(false) => {
                let r = emitter.alloc_reg(Ty(2)); // Bool
                emitter.emit(Instruction::LoadFalse { r_dst: r });
                r
            }
            TypedLiteral::String(_s) => {
                // String constants: emit as normal expr
                expr::emit_expr(&mut emitter, value)
            }
        };
        emitter.emit(Instruction::Ret { r_src: r });
    } else {
        // Non-foldable: emit the full expression
        let r = expr::emit_expr(&mut emitter, value);
        emitter.emit(Instruction::Ret { r_src: r });
    }
    let reg_count = emitter.regs.reg_count();
    let reg_types = emitter.regs.types().to_vec();
    bodies.push(EmittedBody {
        method_def_id: *def_id,
        instructions: emitter.instructions,
        reg_count,
        reg_types,
        source_spans: emitter.source_spans,
        debug_locals: emitter.debug_locals,
    });
}
```

**String literal interning (expr.rs + body/mod.rs):**

The problem: `BodyEmitter` holds `&'a ModuleBuilder` (immutable), but `StringHeap::intern()` requires `&mut self`. The fix is to add an optional mutable string heap reference to BodyEmitter that can be used during body emission.

Option A (simpler): Add a `string_interned: FxHashMap<String, u32>` to BodyEmitter that collects strings during emission, then merge them into the main string heap after all bodies are emitted. This avoids the mutability issue.

Option B (better): Pass `&mut StringHeap` separately through BodyEmitter. Since ModuleBuilder has `pub string_heap: StringHeap` (check if it's pub), we can split the borrow.

Option C (simplest pragmatic fix): In `emit_bodies()`, after emit_all_bodies returns, do a second pass that walks the emitted instructions and replaces `LoadString { string_idx: 0 }` with proper interned offsets. This is ugly but works.

<objective>
Fix MC-01 (callee DefId propagation) and BF-01 (CALL_VIRT contract_idx resolution) by adding `callee_def_id: Option<DefId>` to `TypedExpr::Call`, populating it during type checking, and using it during code emission for correct method_idx and contract_idx in CALL, CALL_VIRT, CALL_EXTERN, and TailCall instructions.

Purpose: The compiler currently emits method_idx=0 and contract_idx=0 for all call instructions emitted through the `emit_expr` path because `extract_callee_def_id_opt()` always returns None. This breaks method dispatch and generic contract specialization at runtime.

Output: TypedExpr::Call carries resolved DefIds; CALL/CALL_VIRT/TailCall emit correct tokens.
</objective>

<task type="auto">
  <name>Task 1: Fix stack overflow by spawning compile pipeline on larger-stack thread</name>
  <files>writ-cli/src/main.rs</files>
  <action>
In `cmd_compile()`, wrap the entire 5-stage pipeline (parse through write) in a `std::thread::Builder::new().stack_size(16 * 1024 * 1024).spawn(...)` call. The thread closure captures `input` and `output` by move. The main thread `.join()` the spawned thread and propagates the result.

Specifically:
1. Move the body of `cmd_compile` (from `let src_owned = ...` through `Ok(())`) into a closure.
2. Use `std::thread::Builder::new().stack_size(16 * 1024 * 1024).spawn(move || { ... })` to run it.
3. The closure returns `Result<(), String>`.
4. After `.join()`, unwrap the outer `Result` (thread panic) and propagate the inner `Result<(), String>`.
5. Handle the `.join()` error case (thread panicked) by returning `Err("compilation panicked".to_string())`.

This is the standard Rust pattern for deep AST recursion (used by rustc, swc, etc.) — increase thread stack rather than rewriting recursion to iteration.

Do NOT change the leaking of the source string (`Box::leak`) — it must happen before the spawn since the closure needs `'static` references. Actually, move the leak into the closure since `src_owned` is already an owned String.

  <name>Task 2: Fix register type blob encoding in serialize.rs</name>
  <files>writ-compiler/src/emit/serialize.rs</files>
  <action>
In the `translate()` function in `serialize.rs`, replace the placeholder register type encoding (line ~269):

```rust
// CURRENT (buggy):
let register_types: Vec<u32> = (0..body.reg_count).map(|_| 0u32).collect();
```

With actual type blob encoding using the body's `reg_types` and the `type_sig::encode_type` function:

```rust
let register_types: Vec<u32> = body.reg_types.iter().map(|ty| {
    let token_for_def = |def_id: crate::resolve::def_map::DefId| -> crate::emit::metadata::MetadataToken {
        builder.token_for_def(def_id).unwrap_or(crate::emit::metadata::MetadataToken::NULL)
    };
    let bytes = crate::emit::type_sig::encode_type(*ty, interner, &token_for_def, &mut blob_heap_clone);
    blob_heap_clone.intern(&bytes)
}).collect();
```

However, `builder` is `&ModuleBuilder` (immutable) in the translate function, and `blob_heap` needs mutable access. The solution is to:

1. Change `translate()` to take `builder: &mut ModuleBuilder` (not `&ModuleBuilder`).
2. For each body's register types, encode using `type_sig::encode_type` with a closure that calls `builder.token_for_def()`.
3. Since `encode_type` needs `&mut BlobHeap` and token_for_def needs `&ModuleBuilder`, split the operation: first encode the bytes, then intern them.
4. Use the `interner` parameter (currently unused — remove the underscore prefix).

Also update the `serialize()` function signature to pass `&mut ModuleBuilder` instead of `&ModuleBuilder`.

And update the call site in `emit/mod.rs` (`emit_bodies`) to pass `&mut builder` to `serialize::serialize`.

Edge cases:
- If `body.reg_types` has fewer entries than `body.reg_count` (shouldn't happen but be defensive), pad with 0x00 (Void) type blobs.
- The token_for_def closure should return `MetadataToken::NULL` for unknown DefIds (same pattern used in collect.rs).

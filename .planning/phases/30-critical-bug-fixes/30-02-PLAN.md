  <name>Task 2: Fix phantom MOV elimination in argument packing</name>
  <files>writ-compiler/src/emit/body/call.rs, writ-compiler/src/emit/body/expr.rs</files>
  <action>
The argument packing code in call.rs (`emit_call`, lines 66-92) and the duplicated packing code in expr.rs (Call arm, lines 219-237) always emits MOVs because `regs.next()` returns a register index past all arg_regs, so `arg_reg != slot_reg` is always true.

**Fix for call.rs `emit_call` (lines 66-92):**

Add an "already consecutive" check before allocating a new block:

```rust
let r_base = if argc == 0 {
    emitter.regs.next()
} else {
    // Check if arg_regs are already in a consecutive block
    let first = arg_regs[0];
    let already_consecutive = arg_regs.iter().enumerate().all(|(i, &r)| r == first + i as u16);
    if already_consecutive {
        first
    } else {
        // Need to repack into a consecutive block
        let r_block_start = emitter.regs.next();
        for i in 0..argc {
            let slot_reg = r_block_start + i;
            let arg_reg = arg_regs[i as usize];
            let arg_ty = emitter.regs.type_of(arg_reg);
            let allocated = emitter.regs.alloc(arg_ty);
            debug_assert_eq!(allocated, slot_reg);
            emitter.emit(Instruction::Mov { r_dst: allocated, r_src: arg_reg });
        }
        r_block_start
    }
};
```

```rust
/// Pack argument registers into a consecutive block starting at r_base.
/// If args are already consecutive, returns their existing base register (no MOV emitted).
/// Otherwise, allocates a new consecutive block and emits MOV instructions.
pub fn pack_args_consecutive(emitter: &mut BodyEmitter<'_>, arg_regs: &[u16]) -> u16 {
    let argc = arg_regs.len() as u16;
    if argc == 0 {
        return emitter.regs.next();
    }
    let first = arg_regs[0];
    let already_consecutive = arg_regs.iter().enumerate().all(|(i, &r)| r == first + i as u16);
    if already_consecutive {
        return first;
    }
    let r_block_start = emitter.regs.next();
    for i in 0..argc {
        let slot_reg = r_block_start + i;
        let arg_reg = arg_regs[i as usize];
        let arg_ty = emitter.regs.type_of(arg_reg);
        let allocated = emitter.regs.alloc(arg_ty);
        debug_assert_eq!(allocated, slot_reg);
        emitter.emit(Instruction::Mov { r_dst: allocated, r_src: arg_reg });
    }
    r_block_start
}
```

Then replace all 7 argument packing sites to call `pack_args_consecutive(emitter, &arg_regs)`.

This eliminates phantom MOVs when arguments are already contiguous (common case for simple calls) and centralizes the packing logic.

**BUG-04 fix (return register):**

In `emit_if` (expr.rs, lines 617-663), the function returns `r_then` and ignores `r_else`. When the if/else is the tail expression of a function body, `emit_all_bodies` uses this register for `Ret { r_src: result_reg }`. If the runtime takes the else branch, `r_then` is uninitialized.

Fix: allocate a shared result register and MOV both branch results into it:

```rust
fn emit_if(...) -> u16 {
    let r_cond = emit_expr(emitter, condition);
    let r_result = emitter.alloc_reg(ty);  // shared result register

    let else_label = emitter.new_label();
    let end_label = emitter.new_label();

    let brf_idx = emitter.instructions.len();
    emitter.emit(Instruction::BrFalse { r_cond, offset: 0 });
    emitter.add_fixup(brf_idx, else_label);

    // Then branch
    let r_then = emit_expr(emitter, then_branch);
    emitter.emit(Instruction::Mov { r_dst: r_result, r_src: r_then });

    let br_idx = emitter.instructions.len();
    emitter.emit(Instruction::Br { offset: 0 });
    emitter.add_fixup(br_idx, end_label);

    // Else branch
    emitter.mark_label_here(else_label);
    if let Some(e) = else_branch {
        let r_else = emit_expr(emitter, e);
        emitter.emit(Instruction::Mov { r_dst: r_result, r_src: r_else });
    } else {
        emitter.emit(Instruction::Nop);
    }

    emitter.mark_label_here(end_label);
    r_result
}
```

This ensures `r_result` holds the correct value regardless of which branch was taken.

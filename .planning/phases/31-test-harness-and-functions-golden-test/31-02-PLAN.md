Purpose: void-return helper function, called from main. Tests CALL + RET_VOID wiring and that the call token is non-zero.

```writ
pub fn greet() {
    // no-op void function
}

    greet();
```

Keep it minimal. `main` calls `greet()` — verifies that the CALL instruction references the correct method token for `greet`.

**Step 3: Write `fn_typed_params.writ`.**

Purpose: int and bool typed parameters, typed return values, multiple-parameter function. Tests that registers carry correct type blobs (BUG-02 was fixed; this locks the correct state).

```writ
pub fn add(a: int, b: int) -> int {
    let result: int = a + b;
    result
}

pub fn is_positive(n: int) -> bool {
    n > 0
}

pub fn main() {
    let x: int = add(3, 4);
    let flag: bool = is_positive(x);
}
```

**Step 4: Write `fn_recursion.writ`.**

Purpose: self-recursive function. Tests that the recursive CALL instruction references the correct self-call metadata token (i.e., the method token for `factorial` is the same at the definition site and the recursive call site).

```writ
pub fn factorial(n: int) -> int {
    if n <= 1 {
        1
    } else {
        n * factorial(n - 1)
    }
}

pub fn main() {
    let result: int = factorial(5);
}
```

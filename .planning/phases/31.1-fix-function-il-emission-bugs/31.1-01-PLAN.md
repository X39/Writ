<task type="auto">
  <name>Task 1: Fix BUG-07 — CALL_INDIRECT emitted for direct function calls</name>
  <files>writ-compiler/src/emit/body/expr.rs</files>
  <action>
The bug is in the call dispatch inside `emit_expr` for `TypedExpr::Call` at lines ~185-272.

Root cause: When the callee's stored type is `TyKind::Func`, the code jumps to `emit_call_indirect` even for calls where `callee_def_id` is `Some(id)` (a statically known named function). A plain `fn add(a, b)` call has a resolved `callee_def_id` but may carry `TyKind::Func` as its type in the type-checked IR.

Fix: Add a guard before the `TyKind::Func` check — if `callee_def_id` is `Some(_)`, skip the delegate path and fall through to the direct-call dispatch regardless of callee type:

```rust
// BEFORE (at line ~185):
if matches!(emitter.interner.kind(callee_ty), TyKind::Func { .. }) {
    let r_delegate = emit_expr(emitter, callee);
    return emit_call_indirect(emitter, expr, r_delegate);
}

// AFTER:
let is_static_call = callee_def_id.is_some();
if !is_static_call && matches!(emitter.interner.kind(callee_ty), TyKind::Func { .. }) {
    let r_delegate = emit_expr(emitter, callee);
    return emit_call_indirect(emitter, expr, r_delegate);
}
// When is_static_call is true, fall through to direct/extern/virtual dispatch below.
```

After this fix, `CALL_INDIRECT` only appears when `callee_def_id` is `None` AND the callee type is `Func` (genuine delegate/closure dispatch). The spurious `r0 ?` delegate register in disassembly also disappears.

The `CallKind::Indirect` arm at line ~266 in the inner match can remain — it is now only reachable in the legitimate delegate path, not for named function calls.

<task type="auto">
  <name>Task 2: Fix BUG-09 and BUG-10 — Void return register and void MOV source in if/else</name>

  <action>
BUG-09 and BUG-10 share a root cause: `emit_expr` returns a void register for block expressions instead of the register holding the final computed value.

**BUG-10 root fix — TypedExpr::Block in expr.rs:**

Locate the `TypedExpr::Block` arm of `emit_expr`. It currently iterates over statements and returns void. Fix: if the block's final element (or a trailing expression) holds a non-void value, return that register.

The typical pattern is:
```rust
TypedExpr::Block { stmts, final_expr, ty, .. } => {
    for stmt in stmts { emit_stmt(emitter, stmt); }
    if let Some(expr) = final_expr {
        emit_expr(emitter, expr)   // return the final expression's register
    } else {
        emitter.alloc_void_reg()
    }
}
```

If the Block variant has a different structure (e.g., a single Vec of TypedStmt/TypedExpr), inspect the actual TypedExpr::Block definition and adapt accordingly. The key invariant: for a non-void block, the register of the last expression must be returned, not a fresh void register.

**BUG-09 fix — mod.rs trailing return:**

After fixing the Block propagation above, `emit_expr(body)` for a function body that is a Block will return the correct final register. The `Ret { r_src: result_reg }` in `emit_all_bodies` then automatically uses the correct register.

Verify the same fix applies to `TypedDecl::Impl` method emission (same pattern, same fix by inheritance from `emit_expr`).

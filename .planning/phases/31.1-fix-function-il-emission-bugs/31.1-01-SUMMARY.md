# Phase 31.1 Plan 01: Fix Function IL Emission Bugs Summary

## Bug Analysis

### BUG-07: CALL_INDIRECT for named function calls
**Root cause:** In `emit_expr` for `TypedExpr::Call`, the `TyKind::Func` check fired unconditionally before checking `callee_def_id`. Plain named function calls may carry `TyKind::Func` as their callee type in the typed IR, so they were incorrectly routed to `emit_call_indirect`.

**Fix:** Added `is_static_call = callee_def_id.is_some()` guard. The delegate path (`emit_call_indirect`) is now only taken when `callee_def_id` is `None` AND the callee type is `TyKind::Func` — i.e., only genuine delegate/closure calls.

### BUG-10: Void MOV source in if/else branches
**Root cause:** The typechecker's block construction (`check_expr.rs`) always sets `tail: None` and places every expression including the final value-producing expression as a `TypedStmt::Expr` in `stmts`. The `emit_expr` Block arm only checked `tail` and fell through to `alloc_void_reg()` for all blocks, returning a void register even when the last statement was a non-void expression.

**Fix:** The Block arm now uses `stmts.split_last()` to check if the last statement is a `TypedStmt::Expr`. If so, all preceding statements are emitted normally, then the final expression is emitted with `emit_expr` (not `emit_stmt`) and its register is returned as the block's value.

### BUG-09: Wrong return register in non-void functions
**Root cause:** Downstream consequence of BUG-10. When `emit_all_bodies` calls `emit_expr(&mut emitter, body)` for a function body (which is a Block), the void register returned by the broken Block arm was stored as `result_reg`. The trailing `Ret { r_src: result_reg }` then referenced a void register.

**Fix:** BUG-10 fix automatically corrects BUG-09. No changes to `mod.rs` were required — once `emit_expr` correctly propagates the block's final register, `result_reg` holds the right value.

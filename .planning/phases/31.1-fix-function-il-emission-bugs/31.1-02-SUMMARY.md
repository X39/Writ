# Phase 31.1 Plan 02: Fix Method Signature Type Ordering and Branch Offset Emission Summary

**One-liner:** Fixed method signature type-ordering (params-first, return-last per spec) in assembler/disassembler, and fixed branch offsets always-zero by propagating LabelAllocator through EmittedBody and applying a 3-pass byte-offset fixup in serialize.rs.

## What Was Built

### Task 1: BUG-08 â€” Method Signature Type Ordering (commit `cdee5e4`)

The disassembler's `decode_method_sig` read the return type before param types, inverting the spec-mandated order. The assembler's `encode_method_sig` and `encode_method_sig_from_params` similarly wrote return type before params. Both were corrected to the spec layout: `param_count:u16 + param_type_blobs + return_type_blob`.

Files changed:
- `writ-assembler/src/disassembler.rs`: `decode_method_sig` â€” params loop runs first, return type read last from remaining bytes; updated doc comment.
- `writ-assembler/src/assembler.rs`: `encode_method_sig` and `encode_method_sig_from_params` â€” param loop moved before return type extend; updated doc comment.

Verification: `cargo test -p writ-assembler` â€” 24 tests pass.

### Task 2: BUG-11 â€” Branch Offsets Always 0 (commit `f5b1682`)

The `LabelAllocator` stored instruction-index positions (not byte positions) because body emission happens before serialization. `apply_fixups` expected byte positions, so all branch offsets were patched with computed-as-0 offsets. The fix propagates the `LabelAllocator` from each `BodyEmitter` through `EmittedBody` to `serialize.rs`, where a 3-pass `encode_instructions` function translates instruction indices to byte positions before applying fixups.

Files changed:
- `writ-compiler/src/emit/body/labels.rs`: Added `resolved_iter()` and `fixups_iter()` accessors.
- `writ-compiler/src/emit/body/mod.rs`: Added `label_allocator: LabelAllocator` field to `EmittedBody`; all 5 construction sites (Fn, Impl, Const, Global, lambda) updated with `label_allocator: emitter.labels`.
- `writ-compiler/src/emit/serialize.rs`: Added `use super::body::labels::{Label, LabelAllocator}`; replaced single-pass `encode_instructions` with 3-pass version; updated call site.
- `writ-compiler/tests/emit_serialize_tests.rs`: Added `label_allocator` field to `EmittedBody` literal in test (Rule 3 auto-fix).

Verification: `cargo build -p writ-compiler` clean; all compiler test suites pass (349 tests total across 9 suites); `BLESS=1 cargo test -p writ-golden` passes all 6 golden tests.

## Key Decisions

1. **3-pass encode_instructions design**: Pass 1 computes byte-start positions for each instruction index (by dry-encoding each instruction). Pass 2 encodes all instructions to bytes. Pass 3 creates a byte-position-keyed `LabelAllocator`, translates resolved labels and fixups from instruction-index to byte-position, then calls `apply_fixups`. This keeps all serialization concern in `serialize.rs` and leaves `LabelAllocator` unchanged.

2. **Accessor design for LabelAllocator**: Rather than exposing the internal `HashMap` and `Vec` directly, added `resolved_iter()` and `fixups_iter()` methods. This keeps the fixup translation logic in `serialize.rs` while providing all data needed without exposing implementation details.

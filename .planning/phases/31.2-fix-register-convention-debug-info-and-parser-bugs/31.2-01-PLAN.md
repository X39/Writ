<objective>
Fix BUG-12: function parameters are never pre-allocated into the register allocator, so each reference to a parameter allocates a fresh uninitialized register. After this fix, emit_all_bodies pre-allocates r0..r(n-1) for each parameter before emitting the body. Re-bless fn_recursion.expected with the corrected 10-register IL.

Purpose: Parameter register stability is a correctness requirement of the IL spec (section 2.16.2). Multiple references to the same parameter must use the same register across all branches.
Output: Updated module_builder.rs, collect.rs, body/mod.rs. Re-blessed fn_recursion.expected. Passing golden tests.
</objective>

```rust
// Pre-allocate parameter registers r0..r(n-1) per IL spec section 2.16.2.
// Parameters are allocated in declaration order before any body emission,
// ensuring all branches of the body find parameters in stable registers.
if let Some(params) = builder.get_fn_params(*def_id) {
    for (name, ty) in params.clone() {
        let r = emitter.alloc_reg(ty);
        emitter.locals.insert(name, r);
    }
}
```

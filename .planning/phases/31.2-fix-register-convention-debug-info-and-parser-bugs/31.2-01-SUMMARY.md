## Decisions Made
- `ast_type_to_ty_simple` uses hardcoded primitive Ty indices (Int=Ty(0)..Void=Ty(4)) without requiring mutable interner access, since `collect_fn` only has `&TyInterner`. Non-primitive types fall back to `Ty(5)` (Error), which is safe since register type affects only debug info, not instruction correctness.
- `params.clone()` in body emitter is intentional: avoids a split-borrow between `builder.get_fn_params` (returns `&Vec`) and `emitter.alloc_reg` (requires `&mut emitter`). The clone is bounded by the parameter count (typically 0-10) and occurs once per function body.

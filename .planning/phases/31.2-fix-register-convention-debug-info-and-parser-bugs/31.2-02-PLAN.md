<task type="auto">
  <name>Task 1: Diagnose and fix the parser to accept bare fn at top level</name>
  <files>writ-parser/src/parser.rs</files>
  <action>
Step 1 — Diagnose. In `writ-parser/src/parser.rs`, look at how `attrs` is defined and whether it can match zero tokens (i.e., returns `Vec::new()` when no `#[...]` attrs are present). If `attrs` can ONLY match one or more attribute lines, change it to `attrs.or(empty().to(Vec::new()))` so it always produces a result even with no attributes.

Also check whether `attrs_vis_decl` uses `.attempt()` or similar backtracking. In chumsky, `.then()` is NOT backtracking by default — if the first parser in a `.then()` chain matches some input but the overall parser fails, it will not retry. Add `.attempt()` to the full `attrs_vis_decl` parser or to key sub-parsers.

Step 2 — Most likely fix. The `attrs` parser probably requires at least one `#[...]` attribute, causing `attrs_vis_decl` to fail when there are no attributes. Change the attrs parser from:

```rust
let attrs = some_attr_parser.repeated()...
```

to ensure it can succeed with zero matches. For example, if `attrs` is defined as `.repeated().collect::<Vec<_>>()`, that already accepts zero matches. If it is `.repeated().at_least(1)` or similar, remove the minimum.

Step 3 — Alternative: If the `attrs` parser is fine but chumsky does not backtrack, add a fallback path. Before `attrs_vis_decl` in the choice, add a dedicated `bare_fn_decl` branch:

```rust
// Direct fn declaration without attrs/vis (private by default)
let bare_fn_item = fn_decl.clone().map_with(|mut fd, e| {
    fd.vis = None; // private by default
    ((cst::Item::Fn((fd, e.span())), e.span()))
});
```

Then in the top-level `item` choice, add `bare_fn_item` before `attrs_vis_decl`:

```rust
let item = choice((
    namespace_decl,
    using_decl,
    impl_item,
    extern_item,
    bare_fn_item,      // NEW: bare fn without attrs/vis
    attrs_vis_decl,    // existing: attrs + optional vis + any declaration
    stmt_item,
))
```

Step 4 — Verify the fix does not break pub fn declarations. Both `pub fn foo() {}` (matched by `attrs_vis_decl`) and `fn foo() {}` (matched by `bare_fn_item` or fixed `attrs_vis_decl`) must work. Test both paths compile correctly.

Step 5 — Handle downstream code. When `fn_decl.vis = None`, the downstream name resolver and collector must treat it as private. Check `writ-compiler/src/emit/collect.rs` `collect_fn` — it reads `entry.vis` from the `DefMap`, not from `fd.vis` directly. The resolver/lowerer maps `fd.vis` to `DefVis`. Confirm `None` visibility is mapped to `DefVis::Private` (not pub). If it currently maps `None` to `Pub` by accident, fix it.

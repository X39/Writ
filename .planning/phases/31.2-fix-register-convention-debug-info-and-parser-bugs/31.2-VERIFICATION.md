## Implementation Details

### BUG-12: Parameter Register Pre-Allocation

**Problem:** Function parameters were never pre-allocated into the register allocator. Each reference to a parameter would allocate a fresh uninitialized register, violating IL spec section 2.16.2 which requires parameter stability across all branches.

**Solution:**
1. Added `fn_param_map: FxHashMap<DefId, Vec<(String, Ty)>>` to ModuleBuilder
2. Populated during collection pass (collect_fn, collect_impl) with parameter names and types
3. Pre-allocation block in body/mod.rs (both Fn and Impl arms) calls builder.get_fn_params() and allocates r0..r(n-1) BEFORE emitting the body
4. Parameters inserted into locals map, ensuring all body references use stable registers

**Verification:** fn_recursion.expected shows r0 as param n (int type), referenced in both if and else branches without fresh allocation.

### BUG-13: Debug Local Names

**Problem:** build_debug_locals hardcoded `name: 0` for all registers, discarding the collected register names from the reg_names map.

**Solution:**
1. Updated TypedStmt::Let (stmt.rs) to push (r_val, name, start_pc, u32::MAX) to debug_locals
2. Updated parameter pre-allocation blocks to push (r, name, 0, u32::MAX) for each parameter
3. Modified build_debug_locals signature to accept `&mut StringHeap`
4. For each register, interns the variable name via string_heap.intern() and stores the returned offset in DebugLocal.name

**Verification:** build_debug_locals code path confirmed; string_heap.intern() called for each named register.

### BUG-14: Byte Offset Source Spans

**Problem:** build_source_spans was using instruction indices as byte offsets, causing incorrect stack trace line numbers. Source spans are recorded as (instr_idx, span) tuples, but SourceSpan.pc must be a byte offset in the encoded instruction stream.

**Solution:**
1. Created compute_instr_byte_starts helper that performs a dry-encode pass to compute byte start offsets for each instruction
2. Updated build_source_spans to accept instr_byte_starts and use it to convert instruction indices to byte offsets
3. Updated translate() call site to compute instr_byte_starts and pass to build_source_spans

**Verification:** compute_instr_byte_starts helper present (lines 408-422); build_source_spans uses it (line 463).

### BUG-15: Parser Bare fn

**Problem:** Parser rejected `fn main() {}` (bare fn without pub/priv) at top level, reporting "found 'Error' at 0..3 expected declaration". The actual root cause was a UTF-8 BOM in fn_empty_main.writ, causing the lexer to emit Token::Error before the parser could attempt to parse.

**Solution:**
1. Removed UTF-8 BOM from fn_empty_main.writ (commit 6f52f18)
2. Parser was already correct (attrs.repeated().collect() accepts zero attributes, visibility.or_not() accepts no visibility)
3. Registered test_fn_empty_main in golden_tests.rs (commit 8678b10)
4. Blessed fn_empty_main.expected with correct IL

**Verification:** fn_empty_main.writ is BOM-free; test_fn_empty_main passes; fn_empty_main.expected exists and is valid IL.

Fix spurious .reg r0 void allocation in empty void function bodies, then re-bless the affected golden files.

Purpose: The block emitter always calls alloc_void_reg() for empty blocks, even when the function returns void and the register is never referenced. This produces a spurious .reg r0 void declaration in the IL output for every empty void function.

Output: Corrected expr.rs that skips register allocation for empty void blocks; re-blessed fn_empty_main.expected and fn_basic_call.expected (both have empty void methods that gain the spurious register today).

<objective>
Fix the runtime VM's CALL instruction handler (and all related method-dispatch instructions) to decode MethodDef metadata tokens before using them as array indices.

Purpose: The compiler emits MethodDef metadata tokens (e.g., 0x07000001 = table_id=7, row_index=1) in the `method_idx` field of CALL, TailCall, SpawnTask, SpawnDetached, and NewDelegate instructions. The runtime dispatch loop was treating these raw token values as direct 0-based array indices into `decoded_bodies`, causing an out-of-bounds crash whenever a compiled .writil file containing function calls was run.

Output: A patched dispatch.rs that decodes tokens to 0-based indices at every method-dispatch instruction site, plus updated vm_tests.rs that use proper MethodDef tokens in Call/TailCall/NewDelegate test cases.
</objective>

pub struct MetadataToken(pub u32);

impl MetadataToken {
    /// Bits 31-24: table ID. Bits 23-0: row index (1-based; 0 = null token).
    pub fn table_id(self) -> u8 { (self.0 >> 24) as u8 }

}

Step 1 — Add token decoding helper to dispatch.rs (at top of execute_one, or inline):

In `writ-runtime/src/dispatch.rs`, add a free function before `execute_one`:

```rust
/// Decode a MethodDef metadata token to a 0-based method body index.
///
/// The compiler emits MethodDef tokens (table_id=7, 1-based row_index) in CALL,
/// TailCall, SpawnTask, SpawnDetached, and NewDelegate instructions. The runtime
/// must strip the table_id byte and convert the 1-based row_index to a 0-based
/// index for decoded_bodies/method_bodies array access.
///
/// Token layout: bits 31-24 = table_id, bits 23-0 = row_index (1-based).
///
/// Example: 0x07000001 → table_id=7, row_index=1 → array_index=0
///           0x07000002 → table_id=7, row_index=2 → array_index=1
#[inline]
fn decode_method_token(token: u32) -> Option<usize> {
    let row_index = token & 0x00FF_FFFF;
    if row_index == 0 {
        None // null token
    } else {
        Some(row_index as usize - 1) // convert 1-based to 0-based
    }
}
```

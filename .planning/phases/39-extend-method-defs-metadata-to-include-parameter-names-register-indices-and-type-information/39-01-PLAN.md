- Note: `fn_param_map` excludes `self` (only regular params); `self` is NOT counted in param_count per research anti-pattern avoidance — see research open question resolution below.

**Research open question resolution (param_count includes self or not):**
The spec §2.16.6 says r0 = self for methods. But `fn_param_map` already excludes self. For consistency with how the VM uses param_count (to know the register layout), param_count should match the number of runtime registers r0..r(param_count-1). For methods with self, that's `fn_param_map.len() + 1`. For free functions, `fn_param_map.len() + 0`. Detect via MethodDefEntry.parent: if parent is Some(_), it's a method (has self unless it's static). Check `flags` bit 1 (is_static) — if NOT static, add 1 for self.

Actually simpler: in `collect.rs`, `fn_param_map` excludes self. The param registers in the body are pre-allocated as: self at r0 (if method), then regular params at r1..r(n). So for a method with `fn_param_map.len() == n`, param_count = n + 1 (for self). For a free function, param_count = n.

This is tracked by checking MethodDefEntry: if `parent.is_some()` AND `!(flags & (1<<1) != 0)` (not static), add 1 for self. But flags contain is_static at bit 1 per `method_flags()`.

Simplest approach: store param_count alongside the method when `add_methoddef` is called. The caller in `collect_fn` and `collect_impl` knows the param count. Pass it as an additional parameter to `add_methoddef` and store in `MethodDefEntry`.

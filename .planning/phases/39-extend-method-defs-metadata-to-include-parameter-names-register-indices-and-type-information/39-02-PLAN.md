From writ-assembler/src/disassembler.rs (current decode_method_sig — to be extended with param names):

The disassembler currently uses `decode_method_sig()` to render `(int, int) -> int` from the signature blob. The function doesn't have access to the calling method's index, so it can't look up ParamDef rows.

The call sites for decode_method_sig are:
1. Top-level method loop (section 7): `decode_method_sig(&module.blob_heap, md.signature, module)`
2. Impl method loop (section 4): same pattern
3. Contract method loop (section 3): same pattern (no ParamDef for contract methods)
4. Extern fn loop (section 6): no params in ParamDef (extern fns not in MethodDef table)

ParamDef lookup strategy:
- `module.param_defs` is a Vec sorted by parent MethodDef index (via the compiler's sort_by_key in finalize).
- However, since ParamDef doesn't have an explicit `parent` field in the binary format (it was sorted implicitly), the disassembler must reconstruct the association by counting: MethodDef row N's params are the ParamDef rows at positions cumulative-param-count[0..N].
- Better: use `sequence` field — params for method N are the ParamDef rows where sequence == 0, 1, 2... in order AND they appear in the ParamDef table at positions corresponding to method N's declared param count.
- Simplest reliable approach: use the MethodDefRow.param_count (now available) to slice param_defs. For method N, its params span `param_defs[sum_of_param_counts[0..N] .. sum_of_param_counts[0..N] + method.param_count]`. Precompute this offset table at the start of disassembly.

From writ-assembler/src/disassembler.rs (decode_method_sig signature):

The cleanest approach is to create a new helper `decode_method_sig_with_names()` that takes the param name slice from ParamDef rows and zips names with types. Or, change the existing signature to accept `param_names: &[&str]` and have callers supply them.

    **Goal:** For each method in the disassembly output, display parameter signatures as `name: type` instead of just `type`.

    **Step 1: Build a param_count offset table at start of disassemble_inner()**

    Add this computation early in `disassemble_inner()`:
    ```rust
    // Precompute cumulative param_def offsets for each MethodDef.
    // method_param_start[i] = index into module.param_defs where method i's params begin.
    let mut method_param_start: Vec<usize> = Vec::with_capacity(module.method_defs.len() + 1);
    let mut running = 0usize;
    for md in &module.method_defs {
        method_param_start.push(running);
        running += md.param_count as usize;
    }
    method_param_start.push(running); // sentinel
    ```

    **Step 2: Add helper to get param names for a method index**

    ```rust
    let get_param_names = |method_idx: usize| -> Vec<&str> {
        let start = method_param_start.get(method_idx).copied().unwrap_or(0);
        let end = method_param_start.get(method_idx + 1).copied().unwrap_or(start);
        module.param_defs[start..end.min(module.param_defs.len())]
            .iter()
            .map(|pd| s(pd.name))
            .collect()
    };
    ```

    **Step 3: Update decode_method_sig to accept param names**

## Tech Debt Aggregated (from Phase VERIFICATIONs)

### Phase 23 — Type Checking
- **TYPE-12 PARTIAL:** Closure capture list stubbed as empty `vec![]` in `check_lambda`. Lambdas that close over outer variables will have empty capture structs at emit time. Formally tracked as DEBT-01.

### Phase 25 — IL Codegen Method Bodies
- **EMIT-11 PARTIAL:** `DESTROY_ENTITY`, `ENTITY_IS_ALIVE`, `GET_OR_CREATE`, `FIND_ALL` dispatch via general CALL (not dedicated IL instructions). Within Phase 25 scope.
- **EMIT-12 PARTIAL:** `ArrayAdd`, `ArrayRemove`, `ArrayInsert` dispatch via CALL (not `ARRAY_ADD`/`ARRAY_REMOVE`/`ARRAY_INSERT`).
- **EMIT-13 PARTIAL:** `IsSome`, `IsOk` not separately confirmed via dedicated instruction test (but other Option/Result instructions confirmed).
- **EMIT-15 PARTIAL:** `DEFER_PUSH` handler_offset was `0` (known limitation from Phase 25); fixed for general case by Phase 28 `emit_defer` patching.
- **Range emission:** `range_type_token()` returns `0` fallback when no cross-module TypeRef registered (deferred from Phase 28; noted info-level).

### Phase 26 — CLI Integration
- **CLI E2E subprocess:** `writ compile && writ run` as an actual CLI subprocess needs manual release-build verification. Debug-build stack overflow was fixed by BUG-01; API-path E2E tests pass.

### Phase 28 — Codegen Bug Fixes
- **contract_idx:0 for multiple contracts:** `call.rs` line ~115 — CallVirt uses `contract_idx: 0` for cases without a registered mapping. Would be wrong for types implementing multiple contracts. Known warning; pre-existing.
- **range_type_token() zero fallback:** Cross-module TypeRef wiring for Range<T> deferred; returns `0` when not registered.

### DEBT-04 Status
Phase 30 removed `extract_callee_def_id_opt`, `convert_token`, `encode_ty`, and `resolve_type` (dead code). The REQUIREMENTS.md checkbox for DEBT-04 (`[ ]`) has not been updated to reflect this partial cleanup. A minor update to REQUIREMENTS.md (and potentially Phase 36 scope) may be warranted.
